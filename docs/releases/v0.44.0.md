# v0.44.0 Release Notes

Artifact v0.44.0 is the Product Surfaces release. It gives the public shell,
discovery journey, Projects, and Docs one coherent Artifact interface from the
first visit through opening editable work.

This release rewrites every non-editor Artifact product route and removes the
leaked shader-debug page. It deliberately leaves editor workflows, inspectors,
canvas chrome, and AI-assisted creation sequencing to their planned releases.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, persistence, AI scope, package export, and font policy remain
unchanged.

## Highlights

- Unified navigation, account entry, footer, password recovery, and route-level
  recovery through responsive Artifact product patterns and UI Foundation
  controls.
- Migrated home, Showcase, examples, and Projects with explicit loading, empty,
  failure, recovery, local/cloud status, and keyboard-operable states.
- Established one Docs pattern across overview, recipes, reference, node detail,
  and the Nodes/effects learning surface, including Foundation search and filter
  controls.
- Added deterministic Product Surface Pattern specimens to the Artifact style
  guide and removed `/debug/shaders` plus its route-specific component and CSS.

## Scope Boundaries

- Included: all current non-editor Artifact routes, their shared shell, and
  their visible loading, empty, error, success, narrow, and wide states.
- Included: focused cross-browser checks for product navigation, password
  recovery, discovery, Projects, Docs, specimens, overflow, and the removed
  debug route.
- Deferred: editor workflow, inspector, and canvas-chrome migration to v0.45
  through v0.47, followed by final conformance and legacy removal in v0.48.
- Deferred: new AI-assisted creation work to v0.49; the existing AI Generation
  composer remains unchanged.

## Validation

Local validation passed during release prep on 2026-07-23:

- `npm run check`
- `npm run build`
- `PLAYWRIGHT_WEB_SERVER_PORT=4199 npm run test:browser:release`
- focused Chromium Product Surface contracts and desktop/mobile visual review
- configured-auth Chromium coverage for the account overlay

The full browser gate scheduled 507 scenarios across Chromium, Firefox,
WebKit, mobile Chromium, mobile WebKit, and Backoffice desktop/mobile Chromium:
456 completed successfully and 51 were intentional skips. Nine existing
editor, AI polling, 3D package, and Foundation overlay scenarios passed on
their available retry. All v0.44 Product Surface scenarios and the configured
account-overlay scenario passed without retry.

Performance notes:

- `npm run perf:node-editor` was not required because v0.44 does not change
  graph traversal, render algorithms, render signatures, thumbnail queues, or
  node-canvas interaction paths. The Showcase thumbnail change only contains a
  rejected-promise cleanup path and does not alter scheduling or pixels.

## Manual QA

- Reviewed home, Showcase, Projects, Docs, and password recovery at desktop and
  mobile widths for hierarchy, focusable commands, clipping, and overflow.
- Reviewed the Artifact Foundation Matrix and deterministic Product Surface
  Pattern specimens, including public-shell account states, route recovery,
  artwork, project-library, Docs navigation, reference, and learning patterns.
- Completed the required manual macOS Safari smoke across home, Showcase,
  Projects, Docs learning/search, and password recovery, including navigation,
  editable links, fields, and keyboard traversal.

## Accepted Risks

- Conformance covers semantics, keyboard focus, native states, geometry,
  overflow, responsive behavior, and representative visual review; it is not a
  complete visual-regression suite or WCAG certification.
- UI compatibility wrappers and legacy aliases remain until the v0.48
  zero-caller conformance gate; editor surfaces remain intentionally split
  across v0.45 through v0.47.
- The build retains the existing Fast Refresh warnings and large Three.js
  vendor-chunk warning; neither is introduced by this product-surface rewrite.
- Nine existing editor, AI polling, 3D package, and Foundation overlay timing
  scenarios required their available retry in the final local browser gate;
  they remain visible in the release evidence and none are in the v0.44 change
  surface.
