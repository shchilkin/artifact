# v0.21.0 Release Notes

Artifact v0.21 is the local font import release. It extends the typography
library from v0.20 so users can bring their own `.ttf`, `.otf`, `.woff`, and
`.woff2` files into browser-only cover work without changing document render
semantics, graph traversal, thumbnail scheduling, or export behavior.

## Highlights

- Added local font import to the shared Font Library picker.
- Added an Imported font section that works through the same picker in Layers
  and Nodes.
- Store text layers with lightweight `artifact-font://...` references while
  keeping font payloads in IndexedDB outside the active `CanvasDocument`.
- Added portable `.artifact.json` and share-link hydration for imported fonts,
  with payloads stripped back out after import.
- Updated text rendering, node thumbnails, graph output, and export paths to
  wait for imported font readiness where the browser supports `FontFace`.
- Added deterministic fallback behavior for missing or unloadable imported font
  payloads so text stays visible.
- Cleaned imported font display labels so file names like
  `Jost-VariableFont_wght.ttf` and `Poppins Bold.otf` read as `Jost` and
  `Poppins` in the UI.

## Validation

Local validation passed on 2026-05-25:

- `npm run check`
- `npm run build`
- `npm run test:browser` across Chromium, Firefox, WebKit, mobile Chromium, and
  mobile WebKit. The full browser gate reported `192 passed, 1 skipped`.
- Focused browser coverage imports a real local `.ttf`, applies it to a text
  layer, verifies a nonblank canvas, reloads, and reselects the imported font
  from IndexedDB-backed local storage.
- Unit coverage verifies imported font references, portable `.artifact.json`
  hydration, payload stripping back out of active documents, missing-font
  fallback boundaries, and imported font label normalization.

Manual validation:

- The user verified local font import and the cleaned imported-font labels
  locally before release prep.

## Accepted Risks

- Imported font management is intentionally minimal in v0.21. Users can import
  fonts and reuse them locally, but delete/favorite/set management is reserved
  for a future typography library pass.
- Browsers may reject malformed font files. In that case Artifact falls back to
  the bundled/system font stack instead of blanking text.
- Full external Google Fonts browsing/import remains a future candidate, not
  part of this release.
- Playwright WebKit is the automated Safari-family gate. A manual macOS Safari
  pass is still recommended before a broad public announcement.
