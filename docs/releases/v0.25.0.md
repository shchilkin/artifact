# v0.25.0 Release Notes

Artifact v0.25 is the Typography Library v3 & Font Policy release. It adds a
Google Fonts import path to the shared Font Library and completes the editable
project-package font policy started in v0.24.

The release keeps document semantics, graph traversal, renderer/export behavior,
thumbnail scheduling, and raster export unchanged.

## Highlights

- Added Google Fonts import from the shared Font Library using either a family
  name or a Google Fonts CSS2 URL.
- Google Fonts imports are stored through the existing imported font asset
  boundary and carry source URL, license, and embedding policy metadata.
- Regular `PACKAGE` export is now license-aware:
  - open-license Google font files can travel in editable `.artifact` packages;
  - unknown local font files stay metadata-only by default.
- Added explicit `PKG+FONTS` export for user-confirmed editable packages that
  include every imported font file.
- Raster `EXPORT` remains pixel-only and does not include font files.
- Docs now distinguish `PACKAGE`, `PKG+FONTS`, raster export, and missing-font
  recovery.

## Accepted Risks

- Google Fonts import uses the public CSS2 stylesheet flow rather than the
  Google Fonts catalog API. Full catalog search and saved font sets remain
  future work.
- Imported Google font files are stored locally after import. Network failure
  during the initial import shows an import error rather than blocking the rest
  of the editor.
- Unknown local font files remain metadata-only in regular packages. Users must
  choose `PKG+FONTS` when they have the right to distribute those files.
- Server-backed share links and account-synced font libraries remain future
  work.
