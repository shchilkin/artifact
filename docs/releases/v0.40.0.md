# v0.40.0 Release Notes

Artifact v0.40.0 is the Shader Authoring And AI Shader Effects release. It adds
first-class procedural shader fills, input-dependent shader effects, editable
GLSL authoring, and validated AI-created shaders to the node editor.

This release expands graph-native texture creation and image processing. It
deliberately does not add shader animation, video export, reusable custom shader
presets, arbitrary JavaScript execution, or a shader marketplace.

Document shader-node records and compatibility normalization changed as
described below. Graph traversal, package export, and font policy remain
unchanged. Renderer/export behavior, thumbnail signatures, and AI scope now
include definition-backed Code Shader and AI Shader nodes.

## Highlights

- Added explicit Shader Fill and Shader Effect roles so procedural shaders can
  either create standalone textures or transform connected artwork.
- Added Code Shader with editable GLSL, manifest-defined controls, compilation
  diagnostics, transparent empty/error output, and contained WebGL execution.
- Added AI Shader Effect generation with browser validation, one repair path,
  durable provenance, explicit local fallback, and refinement of accepted
  results.
- Added dynamic number, boolean, and color controls backed by serializable
  Shader Definitions and per-node Shader Instances.
- Added preview/export parity coverage for backdrop orientation, shader passes,
  WebGL cleanup, legacy migration, and accepted AI refinement.
- Stabilized local release validation by splitting the cross-browser suite
  across bounded fresh dev-server processes.
- Hardened the local AI launcher so occupied IPv4 or IPv6 ports are detected
  before either development server starts.

## Scope Boundaries

- Included: curated shader fills/effects, explicit fill/effect roles, ordered
  palettes, Code Shader authoring, AI Shader generation/repair/refinement,
  dynamic controls, transparent failure behavior, and renderer/export parity.
- Included: prompt limits, idempotency, quota/provider error handling, explicit
  local drafts, node-level lifecycle states, compiler diagnostics, and release
  gate hardening.
- Deferred: shader animation, timeline control, video/sequence export, animated
  thumbnails, and live per-frame document updates.
- Deferred: Save As Preset, reusable custom definition storage, definition
  history/versioning, generation history, AI cost UI, marketplace, and
  community shader sharing.

## Validation

Local validation passed during release prep on 2026-07-11:

- `npm run release:verify`
- `npm run check`
- `npm run build`
- `npm run test:browser:release`
- `npm run test:browser:release -- --grep "refines an accepted AI shader"`

The browser matrix completed with `355 passed, 38 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, and mobile WebKit. Long local runs were split
across fresh dev-server processes, and no failed test required its available
retry in the final release run. The existing 3D preview timeout fallback was
observed without failing its render/export coverage.

Performance notes:

- `npm run perf:node-editor` passed during release prep because v0.40 changes
  node inspectors, shader thumbnails, graph rendering, and output behavior.

## Manual QA

- Verified Shader Fill and Shader Effect connections, transparent empty/error
  output, and generated controls on the node and inspector surfaces.
- Verified Code Shader compilation feedback and backdrop-aware output without
  destabilizing the graph or final output.
- Verified AI Shader create, validate, accepted-result, refinement, and explicit
  local fallback states against the local AI development stack.
- Verified accepted shader output remains visible while a replacement is being
  created and only changes after browser validation succeeds.
- Verified the local AI launcher skips ports already occupied through IPv4 or
  IPv6 and completes its API health and contract checks on the selected port.

## Accepted Risks

- Shader animation remains deferred until time ownership, live-preview cache
  invalidation, animated thumbnails, and video export have explicit performance
  budgets.
- Custom and AI Shader Definitions remain embedded in project documents. A
  reusable preset library and definition versioning need a separate product and
  persistence pass.
- AI Shader availability still depends on configured provider access and quota.
  The deterministic local draft is intentionally limited and requires explicit
  user choice after a recoverable provider failure.
- WebGL-heavy paths can expose browser or dev-server startup flakes during long
  local runs. The release runner isolates test groups and permits one visible
  retry; runtime failures remain contained as transparent shader output.
