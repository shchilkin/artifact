# v0.11 Acceptance Plan

Status: implemented for `v0.11.0-beta.1`.

## Theme

Layer Workflow And Onboarding.

The goal of v0.11 is to make layer mode feel like the fastest path to a
finished cover while keeping node workflows truthful. A user should be able to
start from blank, image, text, example, recipe, or random seed; build a credible
stack-only cover; and understand when graph/node structure is relevant.

## Product Scope

- Layer list hierarchy for graph-area documents.
- Fast layer-row actions for common operations.
- Layer empty states and quick-start actions.
- Stack-only layer presets or recipes.
- Clear handling of controls that are unavailable or owned by node-local state.
- Visible trust in stack preview/export parity.
- Sectioned onboarding guide.
- First-visit "what changed" or guide entry after beta releases.
- Confirmation/recovery around destructive starts.

## Non-Goals

- Do not change graph-area render semantics. Areas stay organizational only.
- Do not add AI generation.
- Do not add backend persistence.
- Do not add a second preview renderer for speed.
- Do not duplicate durable layer controls outside shared control metadata.

## Acceptance Criteria

- A user can build and export a credible stack-only cover without opening nodes.
- Layer mode does not contradict graph organization when a document uses areas.
- Graph areas read as lightweight folders in the layer workflow without changing
  graph traversal or render order.
- Common layer actions are available from the layer list without forcing a trip
  through the node canvas.
- Empty layer states provide direct starts for image, text, fill, noise, effect,
  examples, recipes, and random seed.
- Node-owned or unavailable controls are explained in the UI instead of silently
  disappearing.
- Stack preview and export parity remains explicit and trustworthy.
- Destructive starts require confirmation and preserve a recovery draft.
- The onboarding guide explains canvas, layers, nodes, sources, effects,
  repeaters, export, projects, and examples at a workflow level.

## Suggested Work Order

1. Improve graph-area presentation in the layer list.
2. Add layer-row quick actions for duplicate, mute, rename, delete, and quick
   add.
3. Improve empty states and quick-start actions.
4. Add one or two stack-only layer recipes.
5. Add explanations for node-owned or unavailable controls.
6. Add the sectioned onboarding guide and first-visit guide entry.
7. Add/adjust focused browser smoke coverage for the new starter flow.

## Validation

Required before closing v0.11:

```bash
npm run format:check
npm run lint
npm run typecheck
npm run test
npm run build
npm run test:browser
```

Use `npm run perf:node-editor` if layer-list or preview work changes render or
thumbnail scheduling.

Release validation for `v0.11.0-beta.1` used the required commands above. The
layer-list changes do not alter preview render scheduling or node-editor
thumbnail work, so the opt-in node-editor performance benchmark was not part
of the release gate.

## Manual QA

- Start from blank and create image-first, text-first, noise-first, recipe, and
  random covers.
- Build and export a stack-only cover without entering node mode.
- Open a graph-area document and verify layer folders/areas are understandable
  without implying a render-order change.
- Confirm duplicate, mute, rename, delete, and quick add work from the layer
  list.
- Confirm preview/export composition matches for `1:1`, `4:5`, `9:16`, and
  `16:9`.
- Confirm destructive starts show confirmation and a recovery path.
