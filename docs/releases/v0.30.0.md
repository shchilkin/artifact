# v0.30.0 Release Notes

Artifact v0.30.0 is the Editor Visual Baseline and Design System release. It
adds a curated editor browser baseline, a live internal style-guide route, and a
source-owned UI primitive foundation so future editor work can move with less
guesswork.

This release changes editor confidence, documentation, visual coverage, and the
first layer of shared UI primitives. It deliberately does not redesign the
landing page, split Showcase / How-to, or change renderer, graph traversal,
persistence, document schema, package export, AI scope, or font policy.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added `tests/browser/v030-visual.spec.ts`, covering the blank editor, layer
  row states, graph/node states, Add Library readability, style-guide
  primitives, properties panels, and reduced visual guardrails.
- Added `/docs/style-guide` plus `docs/style-guide.md` and
  `docs/editor-design-system.md` as the design-system source of truth.
- Added source-owned base UI primitives: action buttons, icon buttons, inputs,
  search fields, badges, toolbars, segmented controls, menus, panels, empty
  states, preview frames, sheets, dialogs, floating menus, and tabs.
- Migrated low-risk repeated command/menu chrome in the public nav, editor
  bottom bar, Projects/Presets sheets, layer/node context menus, Add Library,
  empty layer start actions, AI submit, and image replacement to shared
  primitives while preserving editor behavior.
- Added local effect `seedOffset` controls and persistence so stochastic effect
  nodes can vary without changing the document seed.
- Added Figma-style single-node alignment snapping helpers, guides, browser
  coverage, and node-editor performance verification.

## Scope Boundaries

- Included: curated v0.30 editor visual baseline and focused browser coverage.
- Included: project style guide, editor design-system docs, token/primitives
  foundation, and first shared primitive extraction pass.
- Included: effect-node local seed offsets and single-node alignment snapping
  because they were discovered during real v0.30 editor testing and received
  focused coverage.
- Deferred: landing redesign and any new landing release scope.
- Deferred: Showcase / How-to split, recipe routes, and future guided example
  expansion.
- Deferred: broad editor rewrite, inspector rewrite, full component inventory
  completion, Storybook, Chromatic, Fallow integration, server-backed sharing,
  account sync, and command-palette work.

## Validation

Local validation passed during release prep on 2026-06-04:

- `npm run check`
- `npm run build`
- `npm run test:browser`
- `npm run perf:node-editor`
- `npm run test:browser:chromium -- v030-visual.spec.ts`
- `npm run test:browser:chromium -- v029-smoke.spec.ts -g "blank editor"`
- `npm run test:browser:chromium -- generator.spec.ts -g "default document can export from the browser"`

The full browser gate passed with `276 passed, 19 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, and mobile WebKit.

Performance notes:

- `npm run perf:node-editor` passed because v0.30 includes node alignment and
  drag-state work near the node editor.
- Interaction scenarios stayed inside the preferred budget: node drag and graph
  pan had zero long tasks with p95 frame times around `17ms`; the effect slider
  scenario had one `79ms` long task and p95 around `17ms`.
- Initial node-editor load is still heavy and produced existing warnings: one
  draft fallback timeout and missing local Clerk publishable key warnings. These
  are tracked as residual risk rather than new v0.30 interaction regressions.

## Accepted Risks

- The style-guide route is an internal reference surface, not a complete
  Storybook replacement. Remaining component coverage is tracked in
  `docs/editor-design-system.md`.
- Large composed surfaces such as `CanvasPreview`, `NodeCanvas`, `Sidebar`, and
  `BottomBar` still need reduced future specimens even though their behavior is
  covered by browser/render/performance checks.
- Some legacy feature CSS remains by design. v0.30 extracts low-risk primitives
  first and keeps feature-local geometry where changing it would risk editor
  behavior.
- Full golden visual snapshots are still deferred; v0.30 uses deterministic
  browser structure/readability checks and lower-level render tests.
- Node-editor initial load remains heavier than the interaction paths. The
  v0.30 perf gate confirms drag, pan, and slider behavior, but the first-load
  long-task profile remains future performance work.
- Local perf runs without Clerk configuration still report missing publishable
  key warnings. The benchmark completes and editor interactions remain usable.
- Landing, Showcase / How-to, Fallow, server-backed sharing, account sync, and
  command-palette work remain future release tracks.
