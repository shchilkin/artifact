# v0.45 Artifact Editor Workflows

Status: released on 2026-07-23. Delivery is tracked in the closed
GitHub milestone [`v0.45 Artifact Editor Workflows`](https://github.com/shchilkin/artifact/milestone/7).

## Purpose

v0.45 migrates the editor shell, commands, Layers organization, and Add Library
onto explicit Artifact editor patterns before property inspectors and canvas
chrome move.

The release thesis is: the primary editor workflows share one accessible,
responsive control grammar without changing creative state or rendering.

## Scope

- Establish the closed
  [Artifact editor-workflow inventory](../editor-workflow-inventory.md) and
  deterministic specimens.
- Migrate mode navigation, command surfaces, sidebars, bottom commands, layer
  organization, menus, and Add Library.
- Preserve compatibility contracts needed by v0.46 through v0.48.

## Non-Goals

- No property-inspector migration; that is v0.46.
- No node canvas, preview, thumbnail, gallery, or 3D chrome migration; that is
  v0.47.
- No document, graph, history, renderer, persistence, or export change.

## Acceptance Criteria

- Every visible editor workflow in the
  [closed inventory](../editor-workflow-inventory.md) is migrated or assigned
  explicitly to v0.46, v0.47, or v0.49.
- Layer operations preserve document commands, selection, and undo behavior.
- Add Library preserves browse, search, preview, insert, drag, renderer, and
  graph-command behavior.
- Desktop/mobile keyboard, focus, overlay, and overflow conformance pass.

## v0.45.1 Site Navigation Hotfix

v0.45.1 is a bounded patch to the released editor-workflow line. The v0.44
public-shell migration moved the mobile menu trigger onto the shared
`IconButton`; production CSS chunk ordering could then let the primitive's
display rule override the desktop breakpoint while the mobile menu itself
remained hidden.

The patch:

- raises the desktop responsive selector above the shared primitive rule
  without changing the component contract;
- preserves the existing accessible mobile Open/Close menu behavior;
- adds desktop coverage and runs the public-shell contract against a production
  preview in the release gate;
- keeps Add Library keyboard selection stable when programmatic list scrolling
  moves options beneath a stationary pointer, while preserving activation from
  actual pointer movement.

Delivery is tracked by
[#220](https://github.com/shchilkin/artifact/issues/220) and
[#223](https://github.com/shchilkin/artifact/issues/223), and
[#226](https://github.com/shchilkin/artifact/issues/226) in milestone
[`v0.45.1 Site Navigation Hotfix`](https://github.com/shchilkin/artifact/milestone/10).
Authentication, routing, document state, graph behavior, rendering,
persistence, export, Backoffice behavior, and AI scope remain unchanged.
Release notes are prepared in
[`docs/releases/v0.45.1.md`](../releases/v0.45.1.md).
