# v0.47 Artifact Canvas Chrome

Status: in progress. The v0.45 release gate is complete, so implementation may
proceed in parallel with v0.46; the v0.47 release gate remains blocked by v0.46
so public tags stay ordered. Delivery is tracked in GitHub milestone
[`v0.47 Artifact Canvas Chrome`](https://github.com/shchilkin/artifact/milestone/8).

## Purpose

v0.47 migrates graph, preview, thumbnail/gallery, and 3D workspace chrome onto
the Artifact Design System without changing the pixels or interactions they
control.

The release thesis is: Artifact's creative workspaces share one legible visual
grammar while preserving graph, renderer, pointer, export, and camera semantics.

## Scope

- Inventory graph viewport, nodes, edges, ports, areas, CanvasPreview,
  thumbnails/gallery, primitive viewport, and 3D chrome.
- Add deterministic reduced specimens and focused behavior coverage.
- Validate node-editor performance after chrome migration.

The closed C1 surface/state map, approved non-goals, specimen contract, and
semantic invariants live in
[`docs/canvas-chrome-inventory.md`](../canvas-chrome-inventory.md).

## Non-Goals

- No graph traversal, connection, history, or persistence change.
- No renderer, render-signature, cache, export, pointer-math, or camera-state
  ownership change.
- No property-inspector migration.

## Acceptance Criteria

- Every canvas-chrome surface is migrated or an approved non-goal.
- Category-colored selection, output-path contrast, grid readability, and
  node-local event isolation remain intact.
- Preview/export transparency, pointer geometry, thumbnail sizing/caching, and
  primitive camera parity remain intact.
- Focused browser/render and node-editor performance coverage pass.
