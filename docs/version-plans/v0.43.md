# v0.43 Backoffice UI System

Status: planned. Blocked by the v0.42 release gate. Delivery is tracked in
GitHub milestone [`v0.43 Backoffice UI System`](https://github.com/shchilkin/artifact/milestone/2).

## Purpose

v0.43 migrates every Backoffice route and operational state onto UI Foundation
and the Backoffice UI System without importing Artifact workspace patterns.

The release thesis is: Backoffice has a coherent, accessible operational UI
while preserving account, usage, authorization, and API behavior.

## Scope

- Inventory every visible Backoffice surface and state.
- Migrate shell, navigation, route states, overview, account directory, account
  detail, tier/quota mutations, provider usage, and operation recovery.
- Expand the Backoffice `/style-guide` with reduced operational specimens.
- Remove only superseded Backoffice-local helpers and selectors after their
  consumers migrate.

## Non-Goals

- No account, quota, provider-usage, auth, or API redesign.
- No Creative Content in Backoffice.
- No migration of Artifact product or editor surfaces.

## Acceptance Criteria

- Every Backoffice route is migrated or an explicitly approved non-goal.
- Operational forms preserve current validation, conflict, pending, and audit
  semantics.
- Tables and dense operational data remain readable without page-level mobile
  overflow.
- The Backoffice style guide contains the Foundation Matrix and reduced
  specimens for every operational pattern.
