# v0.46 Artifact Inspector System

Status: planned. Blocked by the v0.45 release gate. Delivery is tracked in
GitHub milestone [`v0.46 Artifact Inspector System`](https://github.com/shchilkin/artifact/milestone/6).

## Purpose

v0.46 migrates every property-editing surface onto one Artifact inspector
language while keeping document, graph, renderer, and provider behavior intact.

The release thesis is: ordinary and advanced properties remain dense and
precise while sharing accessible fields, validation, status, and section
patterns.

## Scope

- Establish a closed inspector inventory and field contract.
- Migrate layer properties, common graph inspectors, effect/material/scene
  inspectors, and shader authoring inspectors.
- Treat existing AI Shader work as visual migration only.

## Non-Goals

- No new AI Chat, Context Assistant, shader, effect, material, or scene
  capability.
- No document defaults, validation, history, serialization, graph, render,
  camera, or provider policy change.
- No canvas or viewport chrome migration.

## Acceptance Criteria

- Every property surface is migrated or explicitly assigned elsewhere.
- Dense fields expose accessible labels, status, disabled, locked, error, and
  validation states.
- Live gestures and committed changes preserve existing history behavior.
- Existing shader/provider flows and graph/document/render invariants remain
  green.
