# v0.24.0 Release Notes

Artifact v0.24 is the Project Package & Font Export Policy release. It adds an
editable `.artifact` project package path for larger local work while keeping
raster artwork export pixel-only and preserving document semantics, graph
traversal, renderer/export behavior, and thumbnail scheduling.

## Highlights

- Added a `PACKAGE` action that downloads an editable `.artifact` project
  package.
- `OPEN` now accepts both `.artifact` project packages and existing
  `.artifact.json` documents.
- Project packages include a manifest with document schema version, image
  inventory, font inventory, graph export-target status, and explicit export
  policy fields.
- Package export hydrates imported image payloads so large editable projects can
  move more reliably than URL share links.
- Imported font handling is license-aware by default: packages preserve
  original text plus font metadata, but unknown imported font files are not
  bundled unless a future explicit embedding mode is added.
- Missing imported fonts keep text visible through fallback rendering and leave
  the original text editable so users can replace the font from the Font
  Library.
- User-facing docs now distinguish raster `EXPORT`, portable `SAVE`, and
  editable `PACKAGE`.

## Accepted Risks

- `.artifact` packages currently use an inspectable JSON wrapper. A compressed
  archive/bundle format remains a future candidate if package size or binary
  payload ergonomics require it.
- Unknown imported font files are not bundled by default. Missing fonts reopen
  with fallback rendering until the user replaces the font.
- SVG/path text outlines are intentionally not the default fallback because
  glyph outlines may still be treated as derived font data under some licenses.
- Server-backed share links remain future work; URL share links still have
  practical size limits for large imported payloads.
