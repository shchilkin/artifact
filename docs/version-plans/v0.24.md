# v0.24 Project Package & Font Export Policy

## Summary

v0.24 should make Artifact projects safer to save, move, and share after
v0.22 hardened local assets, v0.21 added imported fonts, and v0.23 improved
node graph construction.

The focus is **editable project portability**, not a new editor surface. The
goal is to define and implement a practical `.artifact` package direction for
large projects while keeping raster artwork export, document semantics, graph
traversal, render/export parity, thumbnail scheduling, and AI scope unchanged.

Font handling is part of this release because editable project export can cross
from artwork output into font software distribution. Raster artwork exports can
remain pixel-only. Editable packages need a clear policy.

## Goals

- Add or prepare a local project package flow that can carry large project
  assets better than URL share links.
- Keep `.artifact.json` and existing local projects working.
- Preserve imported image and font dependency inventories.
- Keep raster image export pixel-only and independent from font licensing.
- For editable project export, preserve original text content and font metadata
  without embedding unknown font files by default.
- Support visual fallback snapshots for text layers when the original font is
  missing.
- Make missing-font recovery explicit: users can replace the font while keeping
  the original text editable.
- Keep package contents inspectable and forward-compatible.

## Non-Goals

- No backend or account-backed project sync.
- No server-backed share links in this slice.
- No Google Fonts catalog import in this slice.
- No new graph node kind.
- No renderer algorithm changes unless package restore exposes a parity bug.
- No durable document schema change unless the package manifest requires a
  documented migration and focused tests.
- No attempt to bypass font licenses by treating path outlines as automatically
  license-free.

## Font Export Policy

Artifact should separate three different outputs:

1. **Raster artwork export**
   - PNG/JPEG/WebP style export is pixels only.
   - Font files are never included.
   - The current visual result is the export contract.

2. **Editable document/project export**
   - Original text content stays in the document.
   - Font family, style, source, PostScript/name metadata, and import identity
     are preserved when available.
   - Unknown user-imported font files are not embedded by default.
   - If the font is missing on reopen, the text remains editable and can be
     reassigned to another font.

3. **Visual fallback snapshot**
   - A package may include a visual text fallback so the project can open close
     to the original look without the font file.
   - Prefer rasterized layer snapshots for conservative portability.
   - SVG/path outlines can be considered only as an explicit mode because glyph
     outlines may still be treated as derived font data under some licenses.
   - Fallback snapshots are not the canonical editable text source; they are
     recovery/preview aids.

Default stance: metadata + original text + no unknown font file embedding.
Embedding a font file requires an allowlisted open license or explicit user
confirmation with a warning that this distributes font software.

## Acceptance Criteria

- Users have a clearer project export path for large editable work than URL
  share links.
- Raster artwork export remains pixel-only.
- Editable project export preserves original text and font metadata.
- Unknown imported fonts are not silently redistributed.
- Missing fonts reopen with a usable visual fallback or clear replacement
  workflow, while original text remains editable.
- Existing `.artifact.json`, local project, share, preview, and raster export
  behavior is not regressed.
- No graph traversal, renderer/export semantics, or thumbnail scheduling change
  is introduced.
- Full release gate passes.
