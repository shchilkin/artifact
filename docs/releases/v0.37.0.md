# v0.37.0 Release Notes

Artifact v0.37.0 is the 3D Material Nodes release. It adds one graph-native PBR
Material node so metallic, chrome, paper, fabric, plastic, pearl, and other
physically inspired surfaces can drive 3D primitive and model scenes.

This release changes the 3D graph pipeline by adding explicit material,
texture-map, and environment inputs. It deliberately does not become a full
material marketplace, text-mask system, mesh editor, ray tracer, animation
system, or broad font-policy release.

Document schema, graph rendering, renderer/export behavior, thumbnail
signatures, and local asset persistence changed to support PBR material nodes,
material texture maps, environment inputs, and multiple resource drops. AI
scope, package export policy, and font policy remain unchanged.

## Highlights

- Added one PBR Material graph node with base/accent color, metalness,
  roughness, clearcoat, relief, grain, anisotropy, and albedo/roughness/
  metalness/normal/alpha texture-map inputs.
- Connected PBR materials now feed 3D primitive and 3D Scene material slots,
  update live previews and node thumbnails, and render through the same graph
  path used by raster export.
- Added explicit Environment Map graph inputs that can light and reflect in 3D
  scenes instead of behaving only like flat backdrops.
- Added multiple resource drag/drop support so compatible image, model,
  material-map, and environment resources can enter the graph in one drop.
- Deferred Three.js loading for blank editor and non-3D Nodes workflows so the
  large 3D vendor chunk is loaded only when a 3D viewport, 3D renderer, or
  environment preview actually needs it.

## Scope Boundaries

- Included: serializable graph material nodes, dedicated material ports,
  material texture-map ports, material-aware render signatures, inspector
  controls that yield to connected graph maps, and focused persistence/asset
  coverage for material texture references.
- Included: shared Three.js PBR material rendering for primitive previews,
  3D Scene/model rendering, graph output previews, thumbnails, and raster
  export.
- Included: environment-map node rendering from imported HDR/EXR assets or
  graph-rendered 2:1 source pixels, with scene lighting/reflection controls.
- Deferred: map scale/rotation controls, channel-packed map authoring, larger
  material example packs, external HDRI/material marketplace workflows, and
  broad Firefox/WebKit WebGL-heavy material coverage.
- Deferred: new text-mask tooling; v0.37 relies on the existing text, font,
  mask, merge, color, and effect graph nodes downstream of the rendered 3D
  result.

## Manual QA

- Verified a PBR Material node can be added from the node Add Library, selected,
  previewed, and edited through the material inspector.
- Verified connected material nodes feed 3D Scene material input ports and
  visibly change the rendered scene compared with the built-in material mode.
- Verified environment-map inputs affect scene lighting/reflection while
  backdrops remain a separate graph input.
- Verified the material path preserves existing downstream composition with
  graph output, merge/color/effect/text/mask workflows, and raster export.

## Accepted Risks

- PBR material authoring starts with one graph node and scalar/map inputs.
  Deeper map transform controls, channel packing, and a large material example
  library remain future workflow polish.
- Heavy 3D material browser coverage is Chromium-first for this release. The
  full browser suite still covers the app across Firefox, WebKit, and mobile
  projects, but deeper WebGL material parity remains future coverage work.
- `three-vendor` remains a large production chunk after the 3D releases, but it
  is no longer on the blank editor or non-3D Nodes startup path. Further
  splitting of Three.js loaders and deeper 3D prefetch policy remains future
  performance polish.
