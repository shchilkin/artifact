# v0.31.1 Release Notes

Artifact v0.31.1 is the Fallow Cleanup Finalization patch release. It finishes
the v0.31 code-quality pass by publishing the trace-validated cleanup, keeping
the PR changed-code audit blocking, and tightening the release notes format for
public release pages.

This release changes quality tooling, CI enforcement, release notes structure,
and verified cleanup. It deliberately does not ship product features or broaden
renderer/editor behavior.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Published the final v0.31 Fallow cleanup pass after PR #72 was merged into
  `development`.
- Kept Fallow CI blocking for the baseline report and PR changed-code audit
  instead of weakening CI.
- Reduced changed-code Fallow findings to zero: no dead-code issues, no
  complexity findings, and no duplication clone groups.
- Reduced baseline duplication to zero duplicated lines, zero clone groups, zero
  clone instances, and zero files with clones.
- Removed the unused `/api/og` endpoint from the current release scope. Dynamic
  OG generation remains a future option for share, showcase, or seeded preview
  pages; the current public OG image remains the static asset.
- Updated release documentation so GitHub Release bodies stay focused on the
  shipped product and architecture scope.

## Scope Boundaries

- Included: patch version bump from `0.31.0` to `0.31.1`.
- Included: merged v0.31 Fallow cleanup from PR #72.
- Included: blocking Fallow changed-code audit with zero introduced findings.
- Included: release-template cleanup for public release bodies.
- Deferred: v0.32 planning and implementation.
- Deferred: dynamic OG generation for share/showcase/seeded pages.
- Deferred: full-health complexity policy for historical baseline hotspots.
- Deferred: landing work, Showcase / How-to work, command palette,
  server-backed sharing, renderer changes, graph traversal changes, export
  changes, persistence changes, document-schema changes, AI scope changes,
  package export changes, and font-policy changes.

- `npm run check`

Firefox, WebKit, mobile Chromium, and mobile WebKit.

Performance notes:

- `npm run perf:node-editor` was not rerun for the patch-only release commit
  because the patch release changes package metadata and documentation after the
  already-validated v0.31 cleanup merge.
- The v0.31 cleanup pass did run `npm run perf:node-editor`; interaction
  scenarios stayed within budget, while initial node-editor load remains a
  documented future performance follow-up.

## Accepted Risks

- Full-health complexity hotspots remain historical baseline findings outside
  the v0.31 changed-code gate. They need a separate policy and refactor plan
  before being treated as a release blocker.
- Dynamic OG generation is intentionally not part of this patch. The current OG
  strategy remains static for the main site.
- npm install previously reported the existing dependency audit state: 11
  vulnerabilities (4 moderate, 7 high). npm audit remediation is outside the
  v0.31.1 Fallow finalization scope and is not part of the current release
  gate.
