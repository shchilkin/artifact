# v0.15 Acceptance Plan

## Theme

Visual Clarity And Cross-Browser Confidence.

The goal of v0.15 is to make the editor easier to read and safer to trust
across browsers without changing document semantics, renderer behavior, graph
traversal, export behavior, or the v0.13 AI alpha scope.

## Product Scope

- Cross-browser Playwright coverage for the main editor flows in Chromium,
  Firefox, and WebKit.
- A clearer dark UI hierarchy for app background, workspace, panels, rows,
  selected states, canvas stage, and node canvas surfaces.
- Better contrast and emphasis for layer rows, active tabs, toolbar actions,
  selected nodes, graph areas, and export/project actions.
- Focused accessibility baseline for text contrast, focus states, and
  state-not-by-color-only checks.
- Documentation updates for release gates and the visual hierarchy rules.

## Non-Goals

- Do not add new AI product scope.
- Do not add backend persistence or account-backed project sync.
- Do not change renderer output, preview/export parity rules, graph traversal,
  node thumbnail scheduling, or primitive camera semantics.
- Do not redesign the product into a light or generic SaaS interface. The UI
  stays dark, raw, mono, and zine-like, but gains clearer visual layers.
- Do not introduce a second preview renderer.

## Acceptance Criteria

- `npm run test:browser` runs the main browser suite in Chromium, Firefox, and
  WebKit plus focused mobile Chromium/WebKit smoke projects.
- CI installs and runs the required Playwright desktop and mobile smoke browser
  projects.
- The editor still passes `npm run check`, `npm run build`, and browser tests.
- Users can distinguish app background, workspace, sidebar/panels, canvas
  stage, layer rows, selected rows, node surfaces, selected nodes, and toolbar
  actions at a glance.
- Layer and node workflows keep the existing behavior and render semantics.
- The UI avoids pure black/white chrome and keeps the warm tinted dark palette.
- Safari/Firefox/mobile manual smoke items are documented for release.

## Manual QA

- Open a stack document in layers mode and confirm the canvas stage is distinct
  from the workspace background.
- Confirm the active layer row, hovered layer row, row actions, and tabs are
  readable without hunting.
- Switch to nodes and confirm node cards, selected nodes, graph areas, handles,
  edges, and the toolbar are visually separated from the grid.
- Export a stack document and a graph document.
- Smoke the same flows in Chromium, Firefox, WebKit, Safari, and a mobile
  viewport before public release.
- Run the focused mobile smoke in Chromium/WebKit when UI layout or shell
  controls change.

## Validation

Required before closing v0.15:

```bash
npm run check
npm run build
npm run test:browser
```

Current status:

- `npm run check` passed after the contrast/mobile-smoke pass.
- `npm run build` passed after the contrast/mobile-smoke pass.
- `npm run test:browser` passed across Chromium, Firefox, WebKit, mobile
  Chromium, and mobile WebKit. Export download-event smoke is covered in
  Chromium/WebKit; Firefox skips that single CI download event because GitHub
  Actions does not report it reliably.
- Observed browser-specific noise is limited to known dev-server console output:
  Vite route-module reload/dynamic-import warnings in WebKit and draft preview
  fallback warnings during render startup. No test failures or behavioral gaps
  were observed.
- AI generation diagnostics are quiet by default and can be enabled with
  `?debug=ai`, `?aiDebug=1`, `?debugAi=1`, `VITE_AI_DEBUG=1`, or
  `localStorage.setItem('artifact-debug-ai', '1')`.

Useful focused commands:

```bash
npm run test:browser:chromium
npm run test:browser:firefox
npm run test:browser:webkit
npm run test:browser:mobile
```
