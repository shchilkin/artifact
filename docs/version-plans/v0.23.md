# v0.23 Graph Add Library Drag

## Summary

v0.23 should make the node editor feel more direct after v0.22 hardened the
local project and portable asset story.

The first slice is **Graph Add Library Drag v2**: users should be able to drag
items from the shared Add Library onto useful graph insertion targets, not only
drop new nodes onto empty canvas. The goal is faster graph construction without
changing graph traversal, renderer/export semantics, thumbnail scheduling,
document schema, or the shared Add Library catalog model.

This is an interaction-quality release, not a new effect/node-kind release.

## Goals

- Keep the existing click-to-add and drag-to-canvas Add Library behavior.
- Add clear insertion affordances for dragging a library item onto an edge.
- Support replacing the current "drop connection on empty canvas" flow with
  the same shared Add Library insertion model where practical.
- Insert compatible layer/effect/source nodes by using existing graph helper
  boundaries, especially edge splitting helpers.
- Preserve Layers and Nodes parity: the Add Library catalog stays shared, while
  graph insertion behavior remains node-canvas specific.
- Keep graph state serializable and undoable through existing document update
  flow.
- Make the drag feedback readable at normal and zoomed-out graph scales without
  corporate/admin chrome.

## Non-Goals

- No new graph node kind.
- No new `CanvasDocument` schema fields.
- No renderer, export, graph traversal, or thumbnail scheduling changes.
- No command palette.
- No account/backend/project-sync work.
- No large Add Library redesign beyond the insertion affordances needed for
  dragging.
- No Layers-panel drag-to-graph behavior in this slice.

## Acceptance Criteria

- Users can drag a compatible Add Library item onto an existing edge and get a
  correctly inserted node.
- Empty-canvas drop, click-to-add, and right-click add menu behavior remain
  intact.
- Inserted graph edges are valid, undoable, and cycle-safe.
- Output preview and export-target rendering stay nonblank after insertion.
- No document schema migration is required.
- Graph traversal, renderer/export output, thumbnail scheduling, and layer
  stack semantics are unchanged.
- Browser behavior remains stable across Chromium, Firefox, WebKit, and mobile
  smoke tests.
