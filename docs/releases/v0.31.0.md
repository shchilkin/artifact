# v0.31.0 Release Notes

Artifact v0.31.0 is the Code Quality and Fallow Integration release. It adds a
read-only codebase intelligence workflow so maintenance risks can be found,
classified, and reviewed before product work picks up again.

This release changes local tooling, CI reporting, release hygiene, and internal
agent guidance. It deliberately does not ship product features, cleanup
deletions, or stricter CI failure gates.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added `fallow@2.88.3` as a dev dependency with report-only package scripts
  for combined baseline, dead-code, duplication, health, and changed-code audit
  reports.
- Added a non-blocking Fallow CI job that runs a baseline report on pushes and a
  changed-code audit on pull requests.
- Captured the first v0.31 Fallow baseline in
  `docs/fallow-v0.31-baseline.md`, including counts, candidate findings,
  cleanup priorities, and the no-delete-without-trace policy.
- Updated `docs/testing.md`, `docs/agent-skills.md`, and `AGENTS.md` with
  agent-safe Fallow commands, JSON/quiet output rules, `--silent` npm usage,
  `|| true` handling, dry-run requirements, trace validation, and telemetry
  boundaries.

## Scope Boundaries

- Included: local report-only Fallow commands and package metadata for
  `fallow@2.88.3`.
- Included: report-only CI wiring with `continue-on-error` while the first
  baseline is reviewed.
- Included: first baseline interpretation and cleanup backlog split into
  actionable, needs-trace, accepted suppression, and false-positive categories.
- Included: release-plan, roadmap, production-readiness, testing, and agent
  documentation updates for the v0.31 workflow.
- Deferred: strict Fallow PR or release gating until baseline findings are
  reviewed and intentional APIs/workspace plumbing are handled.
- Deferred: automatic deletion or auto-fix from Fallow findings.
- Deferred: landing work, Showcase / How-to work, command palette, server-backed
  sharing, renderer changes, graph traversal changes, export changes,
  persistence changes, document-schema changes, AI scope changes, package export
  changes, and font-policy changes.

## Validation

Local release-gate validation passed during release prep on 2026-06-05:

- `npm run check`
- `npm run build`
- `npm run test:browser`

The full browser gate passed with `270 passed, 25 skipped` across
Chromium, Firefox, WebKit, mobile Chromium, and mobile WebKit.

Fallow validation and baseline capture ran during release prep:

- `FALLOW_AGENT_SOURCE=codex npm run --silent fallow --`
- `FALLOW_AGENT_SOURCE=codex npm run --silent fallow:audit -- --base origin/development --gate all`

Performance notes:

- `npm run perf:node-editor` was not required because v0.31 changed tooling,
  CI, package metadata, and internal documentation only. It did not touch React
  Flow wiring, node-editor rendering, high-frequency gestures, thumbnail
  scheduling, or editor behavior.
- Fallow baseline is report-only. The combined baseline reported 118
  dead-code/dependency issues, 176 duplicate clone groups, 228 complexity
  findings, zero circular dependencies, and zero unresolved imports.
- The local changed-code audit fallback used `--gate all` and returned
  `verdict: "fail"` because existing dependency findings for `@artifact/shared`
  and `isbot` are present in changed package manifests. This is documented as
  report-only signal, not a blocking v0.31 release gate. The CI job remains
  non-blocking until those findings are triaged.

## Accepted Risks

- Fallow reports real baseline findings, including unused dependency candidates,
  unused-file candidates, duplicate API script/env shapes, and complexity
  hotspots. They are intentionally documented as backlog, not fixed in v0.31.
- The Fallow CI job is non-blocking. New findings can appear in logs before the
  team decides which suppressions or stricter thresholds are safe.
- `fallow audit --base ...` returned exit code 2 in the dirty local release-prep
  worktree because it could not create a temporary base worktree. The documented
  local fallback is `--gate all`; CI still runs the PR-base command in a clean
  checkout.
- npm install reported the existing dependency audit state: 11 vulnerabilities
  (4 moderate, 7 high). npm audit remediation is outside the v0.31 Fallow
  integration scope and is not part of the current release gate.
