# v0.12 Acceptance Plan

## Theme

Examples, Recipes, And Effect Coverage.

The goal of v0.12 is to turn the v0.11 layer/onboarding foundation into
learnable, regression-tested workflows. Examples should teach what they
demonstrate, docs should explain real recipes, and effect/source range changes
should have focused coverage.

## Dependency On v0.11

v0.12 assumes the v0.11 layer workflow and onboarding paths exist. Some research
and drafting can happen earlier, but final examples, recipes, and docs should
match the v0.11 UI and starter flows.

## Product Scope

- Graph-first recipe starter documents.
- Common-cover recipes: photo plus type, noisy texture plus type, sticker/grid
  motif, primitive over image, and print-damage poster.
- Example categories, used-node summaries, and clearer "start from this" copy.
- Add-node search and grouping improvements for recipes and starters.
- Task-oriented user docs.
- Practical blend-mode guidance.
- Layer-vs-node workflow guidance.
- Effect-family recipes aligned with separated focused effect nodes.
- Troubleshooting docs.
- Effect/source range audit and coverage for changed controls.

## Non-Goals

- Do not start AI generation work in v0.12.
- Do not add backend persistence.
- Do not add broad visual snapshots until GPU/WebGL tolerance is defined.
- Do not add new effect parameters without updating factories, controls,
  renderer, docs, and tests in the same slice.

## Acceptance Criteria

- Examples clearly teach the feature or workflow they demonstrate.
- Each recipe has a clear start path and produces an editable document.
- User-facing docs explain workflows with recipes, not only parameter lists.
- Blend modes are explained through practical cover-making examples.
- Layer-vs-node guidance helps users decide when to stay in layers and when to
  switch to nodes.
- Troubleshooting covers blank previews, missing image assets, browser storage
  limits, GPU/WebGL quirks, and export mismatch.
- Effect/source controls changed in v0.12 have focused render, unit, or browser
  coverage.
- Browser smoke covers at least one layer-first starter path and one docs
  "try this" path.

ready for manual QA.

- Graph-first recipe starter documents for photo plus type, sticker/grid motif,
  primitive over image, and print-damage poster.
- Existing texture/type layer recipe promoted into the examples flow.

  sources/motifs, effect families, troubleshooting, and node catalog.

exports, and hands-on effect/source range checks before tagging v0.12. If the
range audit finds controls that need tuning, update the control metadata,
renderer/docs/tests in the same slice before release.

## Validation

Required before closing v0.12:

```bash
npm run format:check
npm run lint
npm run typecheck
npm run test
npm run build
npm run test:browser
```

Add targeted render/browser tests for any effect or source range changes.

## Manual QA

- Start every new recipe and confirm it creates an editable document.
- Confirm examples show categories, used-node summaries, and useful "start from
  this" copy.
- Follow docs flows from first cover through export.
- Verify changed effect/source ranges at subtle, medium, and destructive values.
- Export representative recipe documents at `1x`, `2x`, and `3x`.
- Check troubleshooting advice against at least one missing-image or blocked
  storage scenario when feasible.
