# v0.45 Artifact Editor Workflows

Status: locally verified release candidate awaiting exact-SHA promotion.
Delivery is tracked in
GitHub milestone [`v0.45 Artifact Editor Workflows`](https://github.com/shchilkin/artifact/milestone/7).

## Purpose

v0.45 migrates the editor shell, commands, Layers organization, and Add Library
onto explicit Artifact editor patterns before property inspectors and canvas
chrome move.

The release thesis is: the primary editor workflows share one accessible,
responsive control grammar without changing creative state or rendering.

## Scope

- Establish the closed
  [Artifact editor-workflow inventory](../editor-workflow-inventory.md) and
  deterministic specimens.
- Migrate mode navigation, command surfaces, sidebars, bottom commands, layer
  organization, menus, and Add Library.
- Preserve compatibility contracts needed by v0.46 through v0.48.

## Non-Goals

- No property-inspector migration; that is v0.46.
- No node canvas, preview, thumbnail, gallery, or 3D chrome migration; that is
  v0.47.
- No document, graph, history, renderer, persistence, or export change.

## Acceptance Criteria

- Every visible editor workflow in the
  [closed inventory](../editor-workflow-inventory.md) is migrated or assigned
  explicitly to v0.46, v0.47, or v0.49.
- Layer operations preserve document commands, selection, and undo behavior.
- Add Library preserves browse, search, preview, insert, drag, renderer, and
  graph-command behavior.
- Desktop/mobile keyboard, focus, overlay, and overflow conformance pass.

## Validation

```bash
npm run check
npm run build
npm run test:browser:release
```

Local release validation passed on 2026-07-23:

- the closed inventory assigns every visible workflow to v0.45 or an explicit
  later release;
- the Artifact style guide contains 48 deterministic editor-workflow
  specimens;
- the complete browser release gate scheduled 538 scenarios across Chromium,
  Firefox, WebKit, mobile Chromium, mobile WebKit, and Backoffice
  desktop/mobile Chromium: 487 passed and 51 were intentional skips;
- focused repeated Firefox and WebKit checks cover Foundation overlays, initial
  style-guide scroll, Add Library option scrolling, import, and export;
- performance validation is not required because graph traversal, renderer
  algorithms and signatures, thumbnail scheduling, and canvas hot paths are
  unchanged.
