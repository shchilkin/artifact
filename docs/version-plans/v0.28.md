# v0.28 Editor Guardrails v2

## Summary

v0.28 continues the editor-confidence track after v0.27. The goal is to make
editing feel more predictable in dense real projects: users should know what is
selected, where it lives, whether it affects export, and which actions are
available before they commit a destructive or structural change.

This is a guardrails and orientation slice, not a renderer or backend slice. It
should preserve document/render/export semantics unless a focused bug fix proves
that existing behavior is already inconsistent.

## Goals

- Clarify lock behavior across Layers and Nodes without surprising graph or
  export changes.
- Make selected-target breadcrumbs more useful: layer stack position, folder or
  area membership where available, and graph output-path context.
- Make hidden, locked, selected output target, graph-only utility, source, and
  effect states read consistently in the properties surfaces.
- Keep `LayerControls` shared between Layers and Nodes.
- Add focused tests for the guardrail rules before broad browser coverage.

## Non-Goals

- No new renderer algorithm.
- No graph traversal or export-path redesign.
- No AI, account, project sync, share-link storage, or backend expansion.
- No corporate/admin UI treatment.
- No broad visual redesign of the editor shell.
- No coverage thresholds unless separately approved after reviewing the v0.27
  baseline.

Release note: v0.28 keeps durable locking layer-backed only. Graph-only utility
nodes remain movable/deletable and show explanatory target context instead of
adding a new document schema field.

## Acceptance Criteria

- Users can tell what is selected and whether it affects export in both Layers
  and Nodes.
- Locked and hidden targets have consistent, visible guardrails.
- Breadcrumb/target context helps in dense projects without adding heavy UI.
- Layers and Nodes keep shared control behavior.
- No render/export semantic change is introduced.
- No schema migration is introduced unless the migration and package/import
  behavior are implemented and tested in the same slice.

## Deferred Candidates

- Server-backed share links.
- Account-synced project and font libraries.
- Visual regression snapshots for editor state.
- Graph-only lock schema if v0.28 chooses not to introduce it.
