# v0.22.0 Release Notes

Artifact v0.22 is the project and asset robustness release. It hardens the
local-first workflow after local font import so documents with imported images,
imported fonts, layers, nodes, export settings, and local project snapshots can
move through save, open, share, reload, and export without silently losing
creative dependencies.

This release preserves document semantics, graph traversal, thumbnail
scheduling, renderer behavior, and export output for documents whose assets are
available.

## Highlights

- Added a shared document asset boundary for save/open/share/project flows.
- Added dependency inventory helpers for imported image refs, imported font
  refs, portable payloads, and graph export-target presence.
- Save and share preparation now hydrates local image/font refs into portable
  payloads where possible, then strips those payloads back out of active editor
  state after import.
- Local project save/load stores portable assets before snapshotting, then
  restores lightweight `artifact-asset://...` and `artifact-font://...`
  references on load.
- Missing imported images now show a clear replacement state instead of a
  technical failure.
- Missing imported fonts keep fallback text visible and surface a readable
  missing-font state in the shared Font Library picker.
- Added browser coverage for portable `.artifact.json` save/open, stack export,
  graph export through the output node, share-link roundtrip, local project
  roundtrip, missing image fallback, and missing font fallback.
- Updated state/testing docs for the portable document boundary and asset
  ownership model.

## Accepted Risks

- Share links still use URL payloads, so very large imported images or real font
  files can exceed practical browser/server URL or header limits. Full real-font
  portability is covered by `.artifact.json` and local project roundtrips; a
  future server-backed share flow or project bundle should handle large payloads.
- The dedicated `.artifact` project bundle remains future work. v0.22 prepares
  the shared portable asset boundary but does not add a new package format.
- Local projects remain browser-local. Account-backed project sync and cloud
  asset storage are still future platform work.
- Playwright WebKit is the automated Safari-family gate. A manual macOS Safari
  pass is still recommended before a broad public announcement.
