# v0.14 Acceptance Plan

## Theme

Editor Beta: Layer Workflow, Onboarding, And Local-First Reliability.

The goal of v0.14 is to ship a parallel editor-focused beta line while the
v0.13 private AI alpha continues separately. This release should make Artifact
feel faster to understand and safer to use as a local-first cover editor, with
no new dependency on AI generation, Clerk, the VPS API, provider keys, or
server-side project sync.

## Release Line

`v0.14.0-beta.1` is the proposed editor/local-first release line.

The v0.13 alpha line remains available for AI operations and reliability fixes:

- `v0.13.0-alpha.2`, `v0.13.0-alpha.3`, and similar patch alphas should stay
  focused on AI access, HTTPS/API smoke, quota/accounting, provider defaults,
  cleanup, and deploy hardening.
- Shared editor bug fixes may be cherry-picked between release branches.
- New AI product scope should not block the v0.14 editor beta.

## Product Scope

- Layer list hierarchy for graph-area documents.
- Fast layer-row actions for common operations.
- Layer empty states and quick-start actions.
- Stack-only layer presets or recipes.
- Clear handling of controls that are unavailable or owned by node-local state.
- Visible trust in stack preview/export parity.
- Sectioned onboarding guide and first-visit guide entry.
- Confirmation/recovery around destructive starts.
- Production-readiness validation for the local editor path.

## Non-Goals

- Do not add new AI generation scope in v0.14.
- Do not require sign-in, Clerk, the VPS API, provider credentials, or server
  storage for the editor beta.
- Do not add backend persistence, server-backed project sharing, billing,
  galleries, or community presets.
- Do not change graph-area render semantics. Areas stay organizational only.
- Do not add a second preview renderer for speed.
- Do not duplicate durable layer controls outside shared control metadata.

## Acceptance Criteria

- A user can build and export a credible stack-only cover without opening
  nodes.
- Layer mode does not contradict graph organization when a document uses areas.
- Graph areas read as lightweight folders in the layer workflow without
  changing graph traversal or render order.
- Common layer actions are available from the layer list without forcing a trip
  through the node canvas.
- Empty layer states provide direct starts for image, text, fill, noise,
  effect, examples, recipes, and random seed.
- Node-owned or unavailable controls are explained in the UI instead of
  silently disappearing.
- Stack preview and export parity remains explicit and trustworthy.
- Destructive starts require confirmation and preserve a recovery draft.
- The onboarding guide explains canvas, layers, nodes, sources, effects,
  repeaters, export, projects, and examples at a workflow level.
- AI controls, if present in the build, remain private/disabled unless the
  existing v0.13 access gates are explicitly configured.

## Current Implementation Status

Started:

- Layer rows now expose direct rename, visibility, duplicate, and delete
  actions with accessible labels, so common edits no longer require opening the
  selected layer controls or relying on double-click rename.
- Empty layer panels now offer direct starts for image, text, fill, noise,
  grain, and examples.
- Blank-canvas recipe starts now expose three layer-first paths: Texture Type,
  Photo Stack, and Noise Poster.
- The Examples page and docs recipes now include the new Photo Type Stack and
  Noise Poster Stack as stack-only layer recipes.
- Graph-area folder headers now read more explicitly as folders and summarize
  contained layer-backed rows plus graph-only helper nodes without changing
  graph traversal or render order.
- Layer controls now explain node-owned or unavailable behavior for primitive
  camera framing, full-canvas noise placement, and graph-area organization.
- Layer rows now expose row-level quick add for stack documents and linear
  export-path graph documents, inserting above the current row without implying
  arbitrary graph rewiring.
- The blank-canvas start prompt links directly to the first-cover guide entry
  in the docs.
- Editor System Cleanup v1 split the layer panel into a feature folder,
  strengthened document command boundaries, and slimmed generator/NodeCanvas
  orchestration without changing render semantics.
- Focused browser coverage verifies row quick actions, empty panel quick
  starts, row quick add, layer-first recipe starts, node-owned/unavailable
  control notes, area folder summaries, node perf toggle persistence, and
  dragging a layer out of an area.
- Local release gate for `v0.14.0-beta.1` has passed with `npm run check`,
  `npm run build`, and full `npm run test:browser`.

Still pending:

- Package version bump, git tagging, and any optional cross-browser manual QA
  before a public announcement.

## Validation

Required before closing v0.14:

```bash
npm run check
npm run build
npm run test:browser
```

Use `npm run perf:node-editor` if layer-list, node-area presentation, preview,
thumbnail, or render scheduling work changes performance-sensitive paths.

## Manual QA

- Start from blank and create image-first, text-first, noise-first, recipe, and
  random covers.
- Build and export a stack-only cover without entering node mode.
- Open a graph-area document and verify layer folders/areas are understandable
  without implying a render-order change.
- Confirm duplicate, mute, rename, and delete work from the layer list. Confirm
  quick add once the row-level or insertion-point affordance lands.
- Confirm preview/export composition matches for `1:1`, `4:5`, `9:16`, and
  `16:9`.
- Save a `.artifact.json` file and reopen it.
- Copy a share link and verify the document loads from `?doc=`.
- Confirm destructive starts show confirmation and preserve a recovery path.
- Smoke AI-disabled/editor-only behavior so private AI controls do not block
  ordinary editor use.
