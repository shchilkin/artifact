# v0.28.0 Release Notes

Artifact v0.28 is the Editor Guardrails v2 release. It continues the editor
confidence work from v0.27 by making selected targets, lock behavior, hidden
state, and graph output context clearer across Layers and Nodes.

The release keeps document schema, graph traversal, renderer/export behavior,
thumbnail scheduling, AI scope, package export, and font policy unchanged.

## Highlights

- Added shared editor guardrail helpers for layer-backed lock behavior, delete
  protection, and layer-stack reorder policy.
- Clarified the lock contract: locked layer-backed targets resist destructive
  delete/reorder actions, while graph-only utility nodes remain movable and
  deletable until a future durable graph-only lock schema is designed.
- Added compact breadcrumbs to the selected target header so Layers and Nodes
  can show stack position, folder/area membership, output target state, and
  output-path context.
- Improved Layers and Nodes properties parity for source, effect, utility,
  output, hidden, locked, and off-output-path states.
- Disabled inline delete for locked layer-backed nodes in the node editor.
- Added focused unit and browser coverage for guardrail policy and target
  breadcrumbs.

## Accepted Risks

- Graph-only locking remains deferred. v0.28 deliberately avoids adding a new
  durable document field without a full schema, migration, package roundtrip,
  and import/export design.
- Visual regression snapshots remain future work.
- Server-backed share links, account-synced projects, and deeper font catalog
  sync remain future work.
