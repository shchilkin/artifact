# v0.29.0 Release Notes

Artifact v0.29 is the Product Surface Recovery release. It makes the public
surface catch up with the editor that already exists: clearer terminology,
`/showcase` instead of legacy examples, stronger docs, blank-editor entry
points, and shared UI primitives for accessible overlays and menus.

The release deliberately keeps the landing redesign out of scope. Landing
exploration notes are preserved in the v0.29 plan, but the shipped slice is
focused on terminology, docs, showcase, navigation, and UI primitives.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Renamed the public examples route to `/showcase`, with a compatibility route
  left behind for old `/examples` links.
- Rebuilt Showcase as an infinite made-in-Artifact wall: curated projects plus
  seeded random entries, with every tile opening an editable project.
- Split showcase from teaching copy. Showcase is now outcome-first; workflows
  and recipes live in docs or future how-to surfaces.
- Reframed public terminology around Artifact as an editor, not a generator.
- Made Open editor the primary public CTA and aligned Open editor / New blank
  canvas behavior around starting from a blank editor.
- Expanded `/docs/nodes` into a searchable, task-oriented docs surface for how
  Artifact works, possible applications, node/effect reference, files, export,
  and fixes.
- Added Random Showcase v2 formula archetypes for image posters, type posters,
  texture/source studies, and print/effect stacks.
- Introduced source-owned shared UI primitives around Radix/shadcn patterns for
  dialogs, floating menus, tabs, and accessible overlay behavior while keeping
  Artifact's visual system intact.
- Added v0.29 browser smoke coverage for public nav, showcase editor handoff,
  docs search/type filtering, primitive surfaces, and node gallery dialog
  behavior.

## Validation

Local validation passed during release prep on 2026-06-03:

- `npm run check`
- `npm run build`
- `npm run test:browser`
- targeted Chromium/Firefox/WebKit coverage for docs recipe handoff, node mute
  menus, layer area menus, random typography, and mobile smoke tests

The full browser gate passed with `252 passed, 19 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, and mobile WebKit.

Performance notes:

- `npm run perf:node-editor` was not required because v0.29 does not change
  graph traversal, thumbnail scheduling, renderer/export behavior, or
  high-frequency React Flow wiring.

## Accepted Risks

- Landing remains deferred. v0.29 intentionally does not claim a final landing
  redesign.
- How-to / recipes are still a future split surface; v0.29 prepares the
  terminology and docs structure but does not ship a separate how-to route.
- Random Showcase v2 has distinct formula archetypes, but reviewed
  agent-generated showcase entries and node-ready random compositions remain
  future work.
- Visual regression snapshots remain future work; v0.29 adds focused browser
  coverage instead.
- Server-backed share links, account-synced projects, and cloud/community
  galleries remain future work.
