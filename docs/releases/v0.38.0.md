# v0.38.0 Release Notes

Artifact v0.38.0 is the Editor UX, Import Safety, And Visual Identity release.
It makes the everyday editor calmer and safer after the recent 3D and material
work, with clearer Layers and Nodes workflows plus deliberate artifact-file
opening.

This release changes editor chrome, local workflow surfaces, import/drop
confirmation, responsive action bars, and the shipped app glyph. It deliberately
does not change server-backed sharing, account sync, project history, renderer
semantics, package export policy, AI scope, or the broader public-site identity.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Polished the compact editor chrome so `/app` focuses on brand, Layers/Nodes
  mode switching, document status, and workspace actions instead of public-site
  navigation.
- Refined Layers and Nodes controls with a clearer aspect-ratio menu,
  responsive bottom actions, calmer layer rows, balanced node handles, grouped
  node toolbar actions, and improved context menus.
- Added safer artifact document opening: artifact files are staged before they
  replace the current canvas, the current work is saved as a recovery copy, and
  users explicitly confirm the replacement.
- Improved drag/drop feedback so images, artifact documents, editable packages,
  models, environment maps, and other resources communicate what will happen
  before import.
- Refreshed the app glyph, favicon, manifest icon, and shared dropdown/action
  primitives while keeping Artifact's dark, square-edged, print-like character.
- Included the Bad Stream block effect nodes that landed after v0.37 as a
  focused visual-effect follow-up.

## Scope Boundaries

- Included: editor navigation, Layers and Nodes toolbar/action polish,
  aspect-ratio menu polish, row and node readability, context-menu behavior,
  drag/drop preview, and artifact-file replacement safeguards.
- Included: local project/share/open action clarity, app glyph/favicon/manifest
  updates, shared dropdown primitive alignment, and the Bad Stream effect
  follow-up.
- Deferred: public landing redesign, whole-site identity refresh, account sync,
  cloud projects, server-backed share links, project history/versioning, and
  autosave history.
- Deferred: deeper 3D material authoring, material packs, external marketplace
  workflows, broad effect batches, and any renderer/export semantic changes.

## Manual QA

- Verified `/app` in Layers and Nodes at desktop and narrow browser widths while
  polishing editor chrome, responsive bottom actions, and mode/status placement.
- Verified Layers aspect-ratio switching, Add menu behavior, empty starts, row
  context menus, and layer-row readability.
- Verified Nodes toolbar grouping, pan/zoom, edge-drop menus, output-path
  controls, handle placement, metrics overlay behavior, and export actions.
- Verified image drops, artifact document replacement confirmation, editable
  package handling, and v0.37-compatible model/environment resource drops.

## Accepted Risks

- The public landing page and broader brand/site refresh remain future work;
  v0.38 intentionally ships the editor-first identity and chrome pass.
- Project autosave history, project versions, restore/compare, and cloud share
  links remain deferred so the local project model can stay simple in this
  release.
- The full browser suite remains a large single-worker cross-browser gate.
  Continued suite stability and CI runtime work remain useful, but v0.38 ships
  with a clean final browser pass.
