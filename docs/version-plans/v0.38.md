# v0.38 Editor UX, Import Safety, And Visual Identity

Status: planned release train on top of v0.37.0. This plan includes the
already-merged Bad Stream node follow-up in PR #79 and the current editor polish
branch. Public release prep, package metadata bumps, release notes, tags, and
GitHub Release publication remain a separate release-prep step after the feature
PRs are merged into `development`.

## Purpose

v0.38 should make Artifact's everyday editor feel calmer, clearer, and safer
after the v0.36 and v0.37 3D/material expansion. The release tightens the
editor chrome, Layers and Nodes workflows, local project status, import/drop
surfaces, and visual identity without changing document schema, graph traversal,
renderer/export semantics, package export, AI quota policy, or account model.

The release thesis is: users can move through Layers, Nodes, local projects,
randomization, opening/sharing, and drag/drop import with a more coherent editor
surface and with safer artifact-file replacement behavior.

## Product Direction

The editor should feel like one composed creative workspace instead of a stack
of isolated panels. v0.38 pulls the recent design critique work into a bounded
release:

- Keep the compact editor navigation focused on mode switching and document
  status, not public-site actions that do not help while editing.
- Make Layers and Nodes controls easier to scan through consistent button
  groups, responsive bottom actions, clearer context menus, and more legible
  row/node states.
- Treat import/open/share surfaces as deliberate workflows: opening an artifact
  document should stage a replacement, save a recovery copy, and ask for
  confirmation instead of silently replacing the current canvas.
- Make drag/drop feedback explicit enough that users understand whether an
  image, artifact document, project package, model, environment map, or other
  resource will be added or reviewed.
- Refresh the visual identity pieces that ship in the app chrome, favicon, and
  manifest while keeping the existing square-edged, dark, print-like Artifact
  character.
- Include the Bad Stream block effect nodes already merged after v0.37 as the
  small visual-effect follow-up in this release train.

## Non-Goals

- No public landing redesign or full whole-app rebrand. v0.38 is editor-first.
- No server-backed share links, account sync, cloud projects, or project
  history/versioning model.
- No document schema migration beyond normalization already required by
  existing v0.37 graph features.
- No renderer/export semantic change. Preview, thumbnails, gallery, and export
  should keep using the canonical renderer paths.
- No broad 3D material follow-up such as map scale/rotation, material packs, or
  marketplace integration.
- No new AI generation scope, quota policy, or backend behavior.
- No release tag or package metadata bump inside the feature PR. Those belong
  to the follow-up release-prep PR.

## Workstreams

### 1. Editor Chrome And Navigation

- [ ] Keep `/app` focused on editor chrome: brand, Layers/Nodes mode switch,
  aspect/layer/project status, and workspace-safe actions.
- [ ] Remove or relocate public-site actions that create noise while editing.
- [ ] Keep the compact nav responsive across desktop and narrow browser widths.
- [ ] Preserve `/app`, `/projects`, docs, showcase, and public navigation route
  compatibility.

### 2. Layers Workflow Polish

- [ ] Improve Layers header alignment, aspect-ratio menu, Add controls, empty
  state, and row readability.
- [ ] Move secondary row actions into context menus where that keeps rows calm.
- [ ] Keep locked/hidden/selected row states visible without adding
  decorative clutter.
- [ ] Keep layer add, insert-above, rename, duplicate, hide/show, lock, delete,
  and reorder behavior compatible with undo, local persistence, and browser
  coverage.

### 3. Nodes Workflow Polish

- [ ] Improve Nodes toolbar grouping, mode/status placement, bottom actions,
  handle balance, and node shell readability.
- [ ] Keep React Flow pan/zoom, node selection, context menu insertion,
  edge-drop add menus, output-path navigation, and performance overlay behavior
  stable.
- [ ] Preserve new v0.37 material/environment/model node behavior while applying
  the editor polish.
- [ ] Keep node-editor performance checks in the release gate because this
  scope touches React Flow chrome and thumbnail/performance surfaces.

### 4. Import, Drop, And Open Safety

- [ ] Stage `.artifact.json` and editable package files before replacing the
  current document.
- [ ] Save the current canvas as a recovery copy before confirming an artifact
  document replacement.
- [ ] Show a clear confirmation surface with file type, aspect, layer count,
  graph presence, and explicit cancel/open actions.
- [ ] Make drag/drop previews distinguish image/layer resource drops from
  artifact document replacement.
- [ ] Preserve multiple-resource drop support from v0.37, including images,
  models, and environment maps.

### 5. Sharing And Local Project Action Clarity

- [ ] Keep "save active project" conceptually separate from share/download
  actions.
- [ ] Group editable document/package/download/share-link actions under Share.
- [ ] Keep Projects access visible without making it compete with primary
  canvas/export actions.
- [ ] Preserve storage warning/status behavior and local project persistence
  tests.

### 6. Visual Identity And Shared Primitives

- [ ] Refresh the app glyph, favicon, manifest icon, and chrome usage without
  changing product naming.
- [ ] Keep shared dropdown/action primitives aligned with the editor design
  system.
- [ ] Update the internal style-guide route so future UI work can inspect the
  new primitives and states.
- [ ] Avoid user-facing copy that explains agent workflow, release planning, or
  QA process.

### 7. Bad Stream Effect Follow-Up

- [ ] Include PR #79's Bad Stream block effect nodes in the v0.38 release notes
  as the already-merged effect follow-up after v0.37.
- [ ] Confirm the effect nodes preserve preview/export parity and do not broaden
  v0.38 into a general effect-batch release.

## Acceptance Criteria

- The editor opens on latest `development` with package metadata still at
  v0.37 until the release-prep PR bumps v0.38.
- Layers and Nodes mode switching remains reliable across desktop and narrow
  browser widths.
- Wide, square, portrait, and mobile-ish viewports preserve document aspect
  ratio in the layer preview.
- Bottom editor actions remain responsive and usable in Layers and Nodes.
- Dropping or opening an artifact document stages the import, keeps current
  work intact until confirmation, and saves a recovery copy before replacement.
- Dropping normal resources still imports them as layers/nodes/assets without
  forcing the artifact replacement confirmation.
- Existing v0.37 PBR material, environment, 3D model, and 3D scene workflows
  remain available after the UI polish branch is rebased.
- Storage status, Projects, Share, Open, Export, Random, undo/redo, Add Library,
  layer context menus, node context menus, and output-path controls keep their
  expected browser-test coverage.
- `docs/roadmap.md` points to this plan as the next release scope, and release
  prep later creates `docs/releases/v0.38.0.md` from
  `docs/release-template.md`.

## Validation

Focused validation during implementation:

```bash
npm run format:check
npm run typecheck
npm run test:browser -- --grep "uploaded images are stored|dropped artifact files stage|portable documents save and reopen|layer preview display preserves wide document aspect ratio"
```

Release gate before tagging v0.38.0:

```bash
npm run release:verify
npm run check
npm run build
npm run test:browser
npm run perf:node-editor
```

Manual QA before release prep is complete:

- Open `/app` in Layers and Nodes at desktop and narrow widths.
- Confirm Layers aspect menu, Add menu, row context menu, and empty starts.
- Confirm Nodes toolbar, bottom actions, pan/zoom, context menus, output path,
  and metrics overlay.
- Drop an image, a `.artifact.json` document, an editable package, and a v0.37
  compatible resource such as a model or environment map.
- Confirm artifact replacement creates a recovery copy and only opens after
  explicit confirmation.
- Export a nonblank document from both Layers and Nodes workflows.

## Follow-Ups

- Full public-site identity refresh remains a future release with its own
  critique/prototype pass.
- Project autosave history, project versions, restore/compare, and server-backed
  share links remain deferred.
- Deeper 3D material authoring and material example packs remain the v0.37
  follow-up track, not hidden v0.38 scope.
- Broader effect batches remain future work; v0.38 only carries the already
  merged Bad Stream follow-up.
