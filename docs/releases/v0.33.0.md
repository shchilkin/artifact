# v0.33.0 Release Notes

Artifact v0.33.0 is the Storage UX And Capability Hardening release. It makes
the local-first editor easier to trust by showing save, recovery, storage,
offline, and browser capability state directly in the editor.

This release changes local status UI, browser capability detection, project
size visibility, conservative PWA install/offline support, and the local
browser release-gate script. It deliberately does not add server-backed
sharing, account sync, broad asset library, export history, project versions,
or hidden service-worker document storage.

Document schema, graph traversal, renderer output semantics, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added a Projects workspace status marker and panel status area that show
  active snapshot state first, with storage usage, recovery copy details,
  offline-app cache state, and capability diagnostics available behind
  `Storage details`.
- Added project-size visibility in the Projects panel so saved local work is
  easier to reason about before deeper asset-library work.
- Renamed the Projects save action to `Save snapshot` so the UI does not imply
  active project overwrite semantics before that model exists.
- Removed the separate local Presets sheet because it duplicated project
  snapshot behavior without a distinct product job. Projects is now the single
  local snapshot surface.
- Added a conservative PWA slice: manifest metadata, production-only service
  worker registration, static asset caching, and an offline fallback page.
- Added a fresh-server browser release command,
  `npm run test:browser:release`, to avoid stale local dev-server runs during
  release prep.
- Added focused unit and browser coverage for storage summaries, capability
  detection, service-worker registration, manifest availability, and the
  Projects workspace status surface.

## Scope Boundaries

- Included: v0.33 package metadata bump.
- Included: active snapshot status and storage/capability status UI through
  Projects.
- Included: recovery copy surfacing through the Projects status marker and
  Projects panel.
- Included: project/recovery size estimates and storage pressure display.
- Included: conservative install/offline support.
- Included: release browser-gate harness stabilization for local release prep.
- Deferred: server-backed share links, cloud project storage, account sync,
  collaboration, broad asset library, export history, named versions, file
  handlers, push notifications, background sync, and broad unused-asset garbage
  collection.
- Deferred: active project id binding, overwrite semantics for project saves,
  and explicit `Save as` / duplicate project behavior. v0.33 Projects remain a
  local snapshot library.
- Deferred: renderer output, graph traversal, persistence schema, package
  export, AI scope, and font-policy semantic changes.

- `npm run check`

Fallow passed the changed-file release gate with zero dead-code findings, zero
complexity findings, and zero duplicated lines.

Performance notes:

- `npm run perf:node-editor` was not required because v0.33 keeps React Flow
  interaction, node-editor gesture paths, thumbnail scheduling, graph traversal,
  document schema, package export, AI scope, and font-policy behavior
  unchanged. The performance instrumentation helper was hardened so measurement
  failures cannot interrupt render or export flows.

## Manual QA

- Confirmed the editor keeps Projects as the single local-workspace entry point
  with a compact status marker.
- Confirmed project save, storage, recovery, offline-app, and browser
  capability details live inside Projects while warning strips appear only when
  attention is needed.
- Confirmed PWA assets are served while service-worker registration remains
  production-only to avoid stale dev/test bundles.

## Accepted Risks

- The PWA slice caches only the app shell/static assets. User documents,
  projects, imported assets, AI requests, and share payloads remain outside the
  service-worker cache and still depend on the existing local stores/network
  paths.
- Broad unused-asset garbage collection is deferred until the Asset Library /
  cleanup model can trace ownership safely.
- Storage estimates depend on browser support for `navigator.storage.estimate`;
  unsupported browsers show known local project sizes and capability warnings
  instead.
