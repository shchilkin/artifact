# v0.36 3D Model Sources And Retro Game Effects

Status: implementation foundation in progress on top of v0.35.0.

## Purpose

v0.36 should let users bring a supported 3D model into the browser, keep it as
an editable source node, render it through a dedicated 3D Scene node with
environment lighting, and finish the output with focused old-game effects:
low-resolution upscale, indexed palette color, stronger dithering, and hard
pixel-edge treatment.

The release thesis is: users can build PS1-era and old-console artwork from
imported 3D model sources, explicit 3D scene lighting, and deterministic retro
finishing effects without flattening the model into an image before
composition.

## Dependency Note

v0.35 is not merged yet. Its worktree currently changes the same surfaces v0.36
will need: document schema, Add Library, node-canvas node construction,
inspector panels, graph rendering, render signatures, docs, examples, and
browser tests.

v0.36 now builds on `v0.35.0`. Early model-source foundation work has been
rebased onto the v0.35 graph/source surface.

## Product Direction

The intended core workflows are:

- Drop a `.glb` file into the editor and get a model source layer/node.
- Route `3D Model -> 3D Scene -> Retro Resolution -> Indexed Palette -> Dither
  -> Output`.
- Use a dedicated Environment Map node for HDRI/EXR/HDR assets, then connect it
  to the 3D Scene node for camera, framing, material mode, exposure, and light
  direction/intensity. Graph previews, gallery, thumbnails, and export must all
  stay in sync.
- Route `3D Scene -> Mask/Transform/Merge` after v0.35 lands.
- Stack imported images, procedural sources, or model branches under the same
  retro finishing pass.

The model source should feel close to Primitive, but not become Primitive. A
Primitive is generated geometry. A 3D Model is an imported asset with its own
file reference, load state, bounds, material behavior, and portability rules.
A 3D Scene is the render context that turns one or more model inputs plus
environment/light/camera settings into a 2D canvas for the rest of the graph.

## Non-Goals

- No broad Asset Library, project versioning, cloud sync, account storage, or
  server-backed model processing.
- No full multi-file glTF package workflow in the first slice. Start with `.glb`
  because it is self-contained. Embedded-data `.gltf` can be considered only if
  it does not introduce sidecar file resolution.
- No rigging, skeletal animation, timeline export, physics, or audio-reactive
  model animation.
- No material editor beyond the minimum needed for a useful model source.
- No replacing Primitive with model import.
- No hidden per-model light controls that bypass graph state. Lighting belongs
  to the 3D Scene node.
- No second renderer for model or scene previews. Preview, thumbnails, gallery,
  and export must use the canonical renderer path or a documented live overlay
  that commits to the same render inputs.
- No non-serializable Three.js objects, DOM nodes, canvases, object URLs, or
  decoded model caches inside `CanvasDocument`.
- No broad effect batch beyond the retro-game gap needed by this release.

## Acceptance Criteria

- Dropping a supported `.glb` file into the editor creates a model source layer
  and, in Nodes mode, a layer-backed `3D Model` node.
- Model payloads are stored outside `CanvasDocument`; the document contains only
  serializable model metadata and an `artifact-model://...` reference.
- Saved local projects and imported/exported portable documents can restore a
  model source when the model payload is available.
- A model source renders in main preview, node thumbnail, node gallery, graph
  target, and export without requiring a flattened image copy.
- Model camera interaction behaves like Primitive camera interaction: local
  gestures stay responsive, committed state is serializable, and export receives
  the same visible framing.
- Unsupported, missing, failed, or oversized model assets show recoverable UI
  states and do not make the renderer output a silent blank image.
- Retro Resolution produces stable nearest-upscaled pixels and preserves export
  density expectations across scale 1/2/3.
- Indexed Palette maps colors to a limited palette deterministically while
  preserving transparent pixels.
- Dither and edge-crush behavior can recreate the core old-game look when
  stacked with existing Pixelate, Posterize, Scanlines, Grain, RGB Split, and
  Duotone/Split Tone effects.
- New effects are available from Add Library, inspector controls, effect docs,
  random/zero behavior where appropriate, and render signatures.
- Existing primitive, image, stack, and graph render tests remain valid after
  the new model source and retro effects land.
