# v0.41.3 Release Notes

Artifact v0.41.3 is the model-drop graph stability patch. While validating the
v0.41.2 missing-model recovery flow in a cloud-backed project, we found that
dropping a GLB/GLTF in Nodes could append it to the end and rebuild the existing
graph as a linear layer stack.

This release changes only model drag-and-drop insertion in Nodes. It does not
change document schema, graph traversal, rendering or export semantics, cloud
persistence, package export, AI scope, or font policy.

## Highlights

- Dropped GLB/GLTF models now appear at the pointer location in Nodes.
- Existing node positions and connections remain unchanged after model import.
- Layers mode retains its existing layer-stack insertion behavior.

## Scope Boundaries

- Included: GLB/GLTF drag-and-drop placement in the node canvas.
- Included: browser regression coverage for graph edges, existing node
  positions, and the imported model position.
- Deferred: recovery of graph layouts already rewritten and saved by an older
  version.
- Excluded: the experimental Artifact Runtime package and renderer work from
  PRs #196-#198. PR #199 moved that experiment off the product release branch;
  `v0.41.3` contains no Runtime package, workspace, or build wiring.

## Validation

Release-candidate validation completed locally on 2026-07-21:

- `npm run release:verify`
- `npm run check`
- `npm run build`
- `npm run test:browser:release`
- `npm run test:browser:chromium -- tests/browser/v036-3d-model-retro.spec.ts`

The focused Chromium file passed with 11 tests. Its first cold run exposed an
existing transient duplicate-React startup diagnostic in the older 3D
render/export smoke; a clean rerun passed, and the full release runner completed
with 364 passed and 47 intentional skips across Chromium, Firefox, WebKit,
mobile Chromium, and mobile WebKit.

Performance notes:

- `npm run perf:node-editor` is not required because the patch changes only the
  completed model-drop command and does not modify per-frame dragging, graph
  traversal, rendering, or thumbnail scheduling.

## Manual QA

- Confirm the release candidate preserves an existing graph when a GLB is
  dropped into the middle of the node canvas.
- Save and reopen the project and confirm the graph layout remains stable.

## Accepted Risks

- Graph layouts already overwritten and saved by an earlier version cannot be
  reconstructed automatically.
- The fix protects future model drops; it does not alter existing documents or
  infer their previous node positions.
- `npm ci` reports 15 inherited dependency advisories (3 low, 3 moderate, and
  9 high). This patch changes no dependency versions; remediation remains a
  separate trace-backed maintenance pass.
