# Triage Labels

The engineering skills use five canonical triage roles. The role name and the
GitHub label are identical in this repository.

| Role | GitHub label | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | A maintainer must evaluate the request. |
| `needs-info` | `needs-info` | Work is waiting for information from the reporter. |
| `ready-for-agent` | `ready-for-agent` | The issue is fully specified and ready for autonomous implementation. |
| `ready-for-human` | `ready-for-human` | The issue requires human implementation or judgment. |
| `wontfix` | `wontfix` | The request will not be actioned. |

Apply only one triage-role label at a time. Product, area, release, and issue
type labels may coexist with the triage role.
