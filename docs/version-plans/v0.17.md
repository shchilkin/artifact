# v0.17 Editor Creative Controls

## Summary

v0.17 focuses on making effect controls feel like creative controls instead of
engineering inputs. This slice is product polish on top of the existing editor:
no document schema migration, no new graph node type, no render/export semantic
change, and no AI scope expansion.

The first shipped scope is **Focused v1: Controls First**. Layers and Nodes both
use the shared effect inspector, so control polish must land once and stay
consistent across both mental models.

## Goals

- Make effect slider values readable through units such as `%`, `px`, `deg`,
  `bands`, and `steps`.
- Move texture, pixel, and print effects toward useful creative ranges in the
  main sliders while keeping manual overrides for extended values.
- Make Grain, Dither, Pixelate, and Riso Shift less destructive by default.
- Treat Pixelate as the first signature whole-image low-resolution workflow
  using the existing focused effect system.
- Keep preview, node thumbnail, graph output, and export paths unchanged.

## Non-Goals

- No new durable fields in `CanvasDocument`.
- No new graph node kind for Pixelate.
- No renderer algorithm changes unless a focused test proves a preview/export
  mismatch.
- No new UX panels, onboarding flows, backend work, accounts, project sync, or
  AI feature expansion.

## Validation

Release validation for `v0.17.0` passed on 2026-05-24:

- `npm run check`
- `npm run build`
- `npm run test:browser` across Chromium, Firefox, WebKit, mobile Chromium, and
  mobile WebKit with `174 passed, 1 skipped`.
- `npm run perf:node-editor`
- `git diff --check`

`npm run perf:node-editor` was run because the shared Add Library added
drag-to-canvas placement in the node editor. Drag, slider, and pan interaction
scenarios had zero long tasks and p95 frame times around `17-18ms`.

## Acceptance Criteria

- Effect controls read clearly through formatted values and useful ranges.
- Grain, pixel, and print defaults are less destructive and immediately usable.
- Pixelate is clearly presented as a low-resolution creative workflow without a
  new schema field or graph node kind.
- Layers and Nodes expose the same control behavior.
- Browser preview/export behavior remains nonblank and unchanged in semantics.
