# v0.27 Editor Confidence And Coverage

## Summary

v0.27 makes the editor feel more trustworthy while adding a measurable testing
baseline. Layers and Nodes are now powerful enough that the next bottleneck is
orientation: users should always know what they selected, whether it affects the
final output, and which controls are safe to use.

This slice is intentionally small and confidence-first. It should not change
document schema, graph traversal, renderer/export semantics, thumbnail
scheduling, AI scope, package/font policy, or the Layers/Nodes mental model.

## Goals

- Make the current editing target explicit in both Layers and Nodes.
- Use a shared target summary model so Layers and Nodes do not describe the same
  layer differently.
- Improve selected target labels, source/effect/output distinction, hidden
  state, and graph-output-path status.
- Make the properties panel easier to scan without adding corporate/admin UI.
- Add a first repeatable Vitest coverage command and document how to interpret
  it.
- Keep coverage as a baseline/audit tool first, not a hard release gate until
  the signal is understood.

## Non-Goals

- No new lock schema field in v1. The existing layer-backed `locked` flag may
  be used for direct stack/graph guardrails; graph-only node locking remains
  deferred.
- No graph traversal or render-output changes.
- No new visual redesign of the editor shell.
- No generic SaaS dashboard patterns, verbose onboarding, or heavy chrome.
- No broad visual snapshot suite in this first slice.

## Acceptance Criteria

- Users can identify the current editing target in Layers and Nodes without
  inferring from row/card styling alone.
- Source, effect, utility, and output targets use consistent language.
- Hidden and graph-output-path status are visible in the relevant properties
  surfaces.
- Coverage can be run locally with stable scripts and documented caveats.
- No document schema migration is required.
- Render/export output stays unchanged.

## Deferred Candidates

- Graph-only locking and stronger schema-backed lock rules beyond the existing
  layer flag.
- Breadcrumbs that include area/folder membership and selected graph branch.
- Coverage thresholds once the baseline is stable and useful.
- Visual regression snapshots for selected rows, node properties, Add Library,
  graph output, and export/preview parity.
