# v0.19 Node Graph Usability

## Summary

v0.19 should make the node editor easier to read and navigate after the v0.18
Add Library work made it easier to create larger graphs. The goal is graph
clarity, not new render behavior: users should understand what path feeds the
final output, where they are in the graph, and how to get back to the important
nodes quickly.

The first slice is **Graph Readability v1**.

## Goals

- Make the active output path visually legible in the node canvas.
- Improve selected-node clarity without changing node semantics.
- Add navigation affordances for common graph recovery moments, such as jumping
  to the output node or fitting the useful graph area.
- Keep Add Library drag-to-canvas behavior intact and consider edge insertion
  only as a small follow-up if it fits the existing graph helper boundaries.
- Preserve Layers and Nodes parity: graph usability improvements must not make
  layer view misleading.

## Non-Goals

- No new `CanvasDocument` fields unless a pure UI preference truly needs
  persistence.
- No new graph node kinds.
- No graph traversal, renderer, export, or thumbnail scheduling changes.
- No large redesign of React Flow wiring.
- No command palette or text workflow work in this release.
- No account/backend/project-sync work.

- [ ] Selection remains clear at normal and zoomed-out scales.
- [ ] Layers -> Nodes -> Layers preview remains nonblank.
- [ ] Add Library drag-to-canvas still works.

## Acceptance Criteria

- Users can visually identify the path feeding the output node in a non-trivial
  graph.
- Users can recover orientation quickly with output navigation.
- Selected nodes read clearly without adding corporate/admin-style chrome.
- Existing graph traversal, render/export output, thumbnails, and document schema
  are unchanged.
- Browser behavior remains stable across Chromium, Firefox, WebKit, and mobile
  smoke tests.

## Next Candidate

`v0.20` should strongly consider **Text Workflow v1**: typography presets,
better font browsing, text layout starts, multi-font cover workflows, and text
effect chains. This is intentionally kept out of v0.19 so graph usability and
type tooling can each get a focused slice.
