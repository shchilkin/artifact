# v0.34 Active Project Save Model

Status: release-ready for `v0.34.0` after maintainer confirmation on
2026-06-11.

## Purpose

v0.34 turns Projects from a snapshot-only local library into an explicit active
project workflow. When a user creates or loads a project, Artifact remembers the
project id outside the document; `Save Project` updates that same local record,
while the contextual `Copy` action creates a new record and makes the copy
active.

This release exists because v0.33 made storage state visible, but the save
model was still semantically incomplete: a user could save a project and still
see a separate recovery/autosave state because the editor was not truly working
inside that saved project.

## Product Direction

Projects should feel like local editable projects, not only one-off snapshots.
The UI should answer three practical questions:

- Is this document currently saved as a project?
- If it is, does `Save Project` update the same saved item?
- If the user wants a branch, where is the contextual copy action without
  confusing it with the primary save path or recovery state?

## Non-Goals

- No server-backed share links, account sync, collaboration, or cloud storage.
- No broad Asset Library, Export History, Versions, or compare/restore UI.
- No renderer/export, graph traversal, package export, AI scope, font-policy,
  or document-schema semantic changes.
- No new project metadata inside `CanvasDocument`.
- No automatic recovery-copy deletion beyond the existing user-facing delete
  action.
- No broader redesign of the editor chrome or node canvas.

## Acceptance Criteria

- Creating a project marks the current editor document as saved in that project.
- Editing an active project changes the visible state to `Unsaved changes`.
- `Save Project` returns the state to `Saved in project` without adding a
  duplicate project card.
- `Copy` creates a separate local project record and makes that copy the
  active project.
- Loading a saved project binds the editor to that project id.
- `/projects` lists saved local projects and recovery copies from the same
  browser project store used by the editor.
- Opening a saved project from `/projects` returns to `/app` with the saved
  project bound as the active project.
- Opening a recovery copy from `/projects` returns to `/app` without binding a
  saved project id.
- Loading recovery, importing a document, opening a docs/share document,
  starting from a starter, or creating a new blank document clears the active
  project binding.
- Active project binding is not serialized into `CanvasDocument`, exported
  `.artifact.json`, share links, or project packages.
- Existing local project records and recovery copies remain compatible.
