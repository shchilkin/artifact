# v0.41.0 Release Notes

Artifact v0.41.0 is the Account Tiers And Backoffice release. It adds explicit
Free, Creator, and Founder access policy, concurrency-safe AI Generation
accounting, provider-cost visibility, and a separate admin-only operations app.

This release changes how provider-backed AI access and spend are governed. It
deliberately does not add billing, self-service upgrades, configurable tier
policy, account suspension controls, or Creative Content inspection.

AI access contracts, server persistence, generation lifecycles, and production
deployment changed as described below. Document schema, graph traversal,
renderer/export behavior, thumbnail scheduling, package export, and font policy
remain unchanged.

## Highlights

- Added explicit Free, Creator, and Founder tiers, monthly Creator allowances,
  audited quota grants, and cross-feature active-operation limits.
- Added atomic AI operation reservations shared by image and shader generation,
  stable denial reasons, idempotent recovery, and preservation of existing
  results when new work is denied.
- Added append-only provider usage events, versioned micro-USD pricing, daily
  provider reconciliation, and a global monthly Safety Budget.
- Added an admin-only backoffice with overview, account search/detail, quota and
  tier controls, usage visibility, reconciliation status, and immutable audit
  records without exposing prompts, generated assets, or project documents.
- Added exact-account Admin bootstrap, Better Auth role enforcement, and a
  separate VPS deployment boundary for the backoffice.
- Added one gated production workflow that verifies a single commit, stages
  Vercel, deploys Coolify at the same SHA, verifies the API contract, and only
  then promotes the web deployment.

## Scope Boundaries

- Included: fixed Free/Creator/Founder policy, monthly Generation accounting,
  audited quota adjustments, operational concurrency limits, usage/cost ledger,
  Safety Budget, Admin API, and backoffice application.
- Included: Better Auth Admin authorization, production domain/CORS/cookie
  configuration, exact-SHA deployment checks, rollback guidance, Bull Board,
  and server-only provider credentials.
- Deferred: billing, checkout, self-service upgrades, Creator access requests,
  account suspension, configurable tier policy, and role-management UI.
- Deferred: Cloudflare Access, backoffice SSR, immutable container images,
  Admin-triggered provider reconciliation, and physical removal of legacy
  entitlement columns.

## Accepted Risks

- Cloudflare Access is deferred. Backoffice data and mutations remain protected
  by Better Auth Admin authorization, but the additional email-gated perimeter
  is a post-v0.41 hardening step.
- The backoffice remains a client-rendered React Router SPA for this release.
  SSR conversion requires a separate Node runtime, cookie-forwarding review,
  hydration tests, and rollback plan.
- Provider reconciliation runs daily and can be invoked through a server-only
  CLI; an audited Admin UI/API trigger remains follow-up work.
- Legacy entitlement columns remain in storage for rollback compatibility but
  no longer influence AI access decisions. Their physical removal requires a
  verified follow-up migration.
- Coolify still builds verified source revisions. Publishing immutable API,
  worker, Bull Board, and backoffice images and deploying by digest is deferred.
- The dependency tree currently reports 14 npm audit advisories. A dedicated
  post-release remediation pass is recorded in the roadmap; no finding is
  hidden with a suppression.
- The full browser matrix is long-running and retains four retry-sensitive
  scenarios. They passed in the release gate, but remain visible rather than
  being suppressed.
