# v0.45.0 Release Notes

Artifact v0.45.0 is the Editor Workflows release. It gives the editor shell,
Layers, and Add Library one accessible, responsive control grammar while
preserving the creative document and rendering behavior underneath.

This release rewrites the primary editor navigation and organization workflows.
It deliberately leaves property inspectors, canvas and preview chrome, final
legacy removal, and new AI-assisted creation to their planned releases.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, persistence, AI scope, package export, and font policy remain
unchanged.

## Highlights

- Unified editor navigation, command groups, bottom actions, feedback, menus,
  and dialogs through source-owned Artifact workflow patterns.
- Expanded Layers keyboard and multi-selection behavior, organization actions,
  contextual commands, reorder behavior, and undo coverage.
- Rebuilt Add Library as a searchable Layers/Nodes combobox and listbox with
  keyboard navigation, previews, insertion, drag behavior, and responsive
  overlays.
- Added 48 deterministic editor-workflow specimens to the Artifact style guide,
  including narrow, overflow, empty, loading, selection, and overlay states.

## Scope Boundaries

- Included: the editor shell, mode navigation, command surfaces, sidebars,
  bottom commands, layer organization, context menus, and Add Library.
- Included: desktop and mobile keyboard, focus, overlay, target-size, nested
  scrolling, and overflow conformance across the supported browser matrix.
- Deferred: property-inspector composition and controls to v0.46.
- Deferred: node canvas, preview, thumbnail, gallery, and 3D chrome to v0.47;
  final conformance and legacy removal remain v0.48, followed by AI-assisted
  creation in v0.49.

## Validation

Local validation passed during release prep on 2026-07-23:

- `npm run check`
- `npm run build`
- `PLAYWRIGHT_WEB_SERVER_PORT=4215 npm run test:browser:release`
- focused repeated Firefox and WebKit style-guide, Add Library, import, export,
  and initial-scroll checks

The full browser gate scheduled 538 scenarios across Chromium, Firefox, WebKit,
mobile Chromium, mobile WebKit, and Backoffice desktop/mobile Chromium: 487
completed successfully and 51 were intentional skips.

Performance notes:

- `npm run perf:node-editor` was not required because v0.45 does not change
  graph traversal, render algorithms, render signatures, thumbnail queues, or
  canvas interaction hot paths.

## Manual QA

- Reviewed the 48 live workflow specimens and editor surfaces at desktop and
  mobile widths for hierarchy, target size, clipping, nested scrolling, and
  overflow.
- Exercised keyboard selection modifiers, layer organization and context
  commands, Add Library search/navigation/insertion, import focus preservation,
  export pending feedback, and overlay dismissal/focus return.

## Accepted Risks

- Conformance covers semantics, keyboard focus, native states, geometry,
  overflow, responsive behavior, and representative visual review; it is not a
  complete visual-regression suite or WCAG certification.
- Compatibility wrappers and legacy aliases remain until the v0.48 zero-caller
  conformance gate.
- The build retains the existing Fast Refresh warnings and large Three.js
  vendor-chunk warning; neither is introduced by this editor-workflow rewrite.
