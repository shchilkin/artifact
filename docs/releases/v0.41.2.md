# v0.41.2 Release Notes

Artifact v0.41.2 is the missing 3D asset recovery patch. It follows the
v0.41.1 project-package portability fix and addresses the legacy packages that
were exported before model payloads were embedded.

This release changes model error reporting and the model inspector recovery
path. It does not change the document schema, graph traversal, scene rendering
semantics, or the package format.

## Highlights

- A missing local model payload now reports `Model asset missing` instead of
  the misleading `Model load failed`; invalid GLB data keeps the load-failure
  state.
- The model inspector can replace or reattach a GLB while preserving the model
  layer ID, graph connections, downstream effects, and scene settings.
- The selected GLB is stored in IndexedDB under a new stable
  `artifact-model://` reference and becomes available to the live viewport and
  subsequent project exports.
- Existing package v1/document schema v3 files remain supported, and the
  v0.41.1 embedded GLB/EXR clean-context round trip remains covered.

## Scope Boundaries

- Included: truthful missing-versus-invalid model states.
- Included: in-place recovery of a missing imported GLB from the model node
  inspector.
- Included: preservation of the existing graph topology and scene settings
  during recovery.
- Deferred: reconstructing GLB or EXR bytes that an older package never
  contained; the original file is still required once.
- Deferred: model-specific decoder or extension compatibility without the
  original failing asset.

- `npm run build`
- `npm run test:browser:release`
- focused Chromium regressions for missing model reporting, in-place GLB
  recovery, stable IndexedDB-backed references, and graph-edge preservation
- the existing Chromium export, clean browser context, import, IndexedDB
  resolution, and rendered 3D Scene regression for embedded GLB and EXR assets
- Fallow changed-code audit with no introduced dead code, complexity findings,

The local browser gate passed with 363 tests and 45 intentional skips across
Chromium, Firefox, WebKit, mobile Chromium, and mobile WebKit.

Performance notes:

- `npm run perf:node-editor` was not required because the patch does not change
  graph traversal, rendering algorithms, thumbnail scheduling, or node-canvas
  hot paths.

## Manual QA

- Imported a package into a browser context with empty storage and confirmed
  that its unresolved model reports the missing-asset state.
- Reattached a valid GLB from the existing model inspector, confirmed a new
  stable `artifact-model://` reference in local storage, preserved the original
  model-to-scene edge, and observed a rendered model frame.
- Confirmed that a newly exported package still restores both embedded GLB and
  EXR payloads in a clean browser context without selecting files again.

## Accepted Risks

- Older packages that omitted model bytes cannot recreate them; users need the
  original GLB once to repair and re-export the project.
- The originally reported `skull.glb` was unavailable for direct parser
  testing. A separate model-specific failure may still require that source
  asset to reproduce.
