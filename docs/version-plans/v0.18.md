# v0.18 Add Library Workflow

## Summary

v0.18 continues the editor menu work started in v0.17. The goal is to make the
shared Add Library feel like a fast creative command palette for both Layers
and Nodes, without changing document semantics, graph traversal, renderer/export
behavior, thumbnail scheduling, or AI scope.

The first slice is **Library Navigation v1**: faster browse paths for a growing
effect catalog.

## Goals

- Keep Layers and Nodes on one shared Add Library model.
- Make long effect/source menus easier to scan through explicit group filters.
- Let users pin frequently used library items as local favorites.
- Preserve search, recent items, popular starts, recipe groups, and
  renderer-backed previews.
- Keep node drag-to-canvas placement intact.

## Non-Goals

- No new `CanvasDocument` fields.
- No new graph node kinds.
- No renderer or export semantic changes.
- No account/backend/project-sync work.
- No corporate/admin menu redesign.
