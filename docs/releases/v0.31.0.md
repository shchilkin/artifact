# v0.31.0 Release Notes

Artifact v0.31.0 is the Code Quality and Fallow Integration release. It adds a
codebase intelligence workflow and uses the first Fallow reports to remove
trace-validated duplication and dead-code candidates before product work picks
up again.

This release changes local tooling, CI reporting, release hygiene, internal
agent guidance, and verified cleanup inside existing code paths. It deliberately
does not ship product features or broaden renderer/editor behavior.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added `fallow@2.88.3` as a dev dependency with package scripts
  for combined baseline, dead-code, duplication, health, and changed-code audit
  reports.
- Added a blocking Fallow CI job that runs a baseline report on pushes and a
  changed-code audit on pull requests.
- Captured the first v0.31 Fallow baseline in
  `docs/fallow-v0.31-baseline.md`, including counts, candidate findings,
  cleanup priorities, and the no-delete-without-trace policy.
- Reduced the final Fallow report to zero issues and zero duplication:
  `duplicated_lines: 0`, `clone_groups: 0`, `clone_instances: 0`, and
  `files_with_clones: 0`.
- Consolidated repeated browser, render, worker, storage, graph, layer, API, and
  test helpers while removing trace-validated unused files.
- Updated `docs/testing.md`, `docs/agent-skills.md`, and `AGENTS.md` with
  agent-safe Fallow commands, JSON/quiet output rules, `--silent` npm usage,
  `|| true` handling, dry-run requirements, trace validation, and telemetry
  boundaries.

## Scope Boundaries

- Included: local Fallow report commands and package metadata for
  `fallow@2.88.3`.
- Included: blocking Fallow CI wiring for the baseline report and PR
  changed-code audit after the cleanup reduced changed-code findings to zero.
- Included: first baseline interpretation and cleanup backlog split into
  actionable, needs-trace, accepted suppression, and false-positive categories.
- Included: verified Fallow cleanup that removes duplicated lines, unused-file
  candidates, and repeated helper logic without changing product behavior.

  persistence changes, document-schema changes, AI scope changes, package export
  changes, and font-policy changes.

## Validation

Local release-gate validation passed during release prep on 2026-06-05:

- `npm run check`
- `npm run build`
- `npm run test:browser`
- `npm run perf:node-editor`

The full browser gate passed with `270 passed, 25 skipped` across
Chromium, Firefox, WebKit, mobile Chromium, and mobile WebKit.

Fallow validation and baseline capture ran during release prep:

- `FALLOW_AGENT_SOURCE=codex npm run --silent fallow --`
- `FALLOW_AGENT_SOURCE=codex npm run --silent fallow:audit -- --base origin/development`

Performance notes:

- `npm run perf:node-editor` passed after rerunning outside the filesystem
  sandbox because the benchmark needs to bind `127.0.0.1:4174`.
- The interaction scenarios stayed within budget: node drag and graph pan had
  zero long tasks with p95 frame times around `17.5ms`; the effect slider had
  zero long tasks with p95 around `17.4ms`.
- Initial node-editor load still reported 8 long tasks and thumbnail/GPU startup
  work. That remains an accepted performance follow-up, not a new v0.31
  regression.
- The first Fallow baseline showed 3,863 duplicated lines across 176 clone
  groups. The final release-prep report is zero for issues, duplicated lines,
  clone groups, clone instances, files with clones, and duplication percentage.
- The final changed-code audit returned `verdict: "pass"` with zero dead-code
  issues, zero complexity findings, and zero duplication clone groups.

## Accepted Risks

- Full-health complexity hotspots remain historical baseline findings outside
  the v0.31 changed-code gate. They need a separate policy and refactor plan
  before being treated as a release blocker.
- `fallow audit --base ...` can require an unrestricted local environment
  because Fallow creates a temporary base worktree. The final release-prep audit
  passed when run with that permission, and CI runs the same PR-base command in
  a clean checkout.
- The node-editor performance benchmark still shows initial-load long tasks and
  Vite dependency optimization noise in local dev-server output. Interaction
  scenarios stayed within budget.
- npm install reported the existing dependency audit state: 11 vulnerabilities
  (4 moderate, 7 high). npm audit remediation is outside the v0.31 Fallow
  integration scope and is not part of the current release gate.
