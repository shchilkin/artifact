# v0.45.1 Release Notes

Artifact v0.45.1 is the Site Navigation Hotfix release. It restores the
responsive public-shell boundary so the mobile menu trigger no longer appears
on desktop production builds.

This release changes the public navigation CSS cascade and its production
browser coverage. It deliberately does not change authentication, routing,
editor workflows, or creative behavior.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, persistence, AI scope, package export, and font policy remain
unchanged.

## Highlights

- Keeps the mobile-only menu trigger hidden at desktop widths even when
  production CSS chunks load in a different order.
- Preserves the accessible Open/Close menu behavior on narrow layouts.
- Adds the responsive public shell to the production-preview browser release
  gate so bundle-order regressions are caught before deployment.

## Scope Boundaries

- Included: the public Site Navigation responsive CSS boundary.
- Included: focused desktop and mobile production-preview regression coverage.
- Deferred: property-inspector migration, canvas chrome, final UI conformance,
  and AI-assisted creation remain in v0.46 through v0.49.
- Deferred: no unrelated editor, Backoffice, authentication, or routing
  changes.

## Validation

Local validation passed during release prep on 2026-07-23:

- `npm run release:verify -- --version 0.45.1`
- `npm run check`
- `npm run build`
- `npm run test:browser:release`
- `PLAYWRIGHT_WEB_SERVER_PORT=4193 npm run test:browser:release -- --project=chromium v044-product-surfaces.spec.ts --grep "public shell"`

The full browser gate scheduled 542 scenarios across Chromium, Firefox, WebKit,
mobile Chromium, mobile WebKit, and Backoffice desktop/mobile Chromium: 491
completed successfully and 51 were intentional skips.

Performance notes:

- `npm run perf:node-editor` was not required because this patch changes no
  graph traversal, renderer algorithm, render signature, thumbnail queue, or
  canvas interaction hot path.

## Manual QA

- Verified the production preview at 1440 px: desktop navigation remains
  visible and the mobile menu trigger is hidden.
- Verified the production preview at 390 px: the mobile trigger opens and
  closes the accessible menu, updates `aria-expanded`, and closes after
  navigation.

## Accepted Risks

- The focused production-preview checks cover the responsive shell contract,
  not a complete visual-regression suite or WCAG certification.
- The build retains the existing Fast Refresh warnings and large Three.js
  vendor-chunk warning; neither is introduced by this patch.
