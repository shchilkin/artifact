# v0.35 Masks, Rotated Repeats, And Line Fields

Status: release ready.

## Purpose

v0.35 is a focused visual-foundation release. It should unlock mask-driven
branch composition, post-mask branch transforms, repeated motifs with per-copy
internal rotation, and editable warped line material without turning into a
broad editor redesign or effect batch.

The release thesis is: users can build patterned type, circular token systems,
noise-masked merges, and warped line posters from serializable graph nodes and
procedural sources that render identically in preview, thumbnails, gallery, and
export.

## Product Direction

Artifact should keep the existing layer and node workflow, but add three
missing primitives that make advanced poster recipes practical:

- Mask a source branch with a separate alpha, luma, or threshold matte.
- Move, scale, rotate, and fade completed graph branches after masks or effects.
- Build layered dirty shadows from masked or transformed branch alpha.
- Rotate each repeated copy deterministically while preserving existing Repeat
  defaults.
- Generate line fields as editable source material instead of requiring a
  flattened imported image or one-off effect.

The intended core workflows are:

- `Pattern/Text/Image -> Mask -> Merge -> Output`
- `Pattern/Text/Image -> Mask -> Transform -> Merge -> Output`
- `Image/Text -> Mask -> Grime Shadow -> Merge -> Output`
- `Line Field -> Mask -> Repeat -> Output`
- `Line Field -> Text/Effects -> Output`
- `Texture -> Mask(noise) -> Merge -> Output`

## Non-Goals

- No broad editor redesign.
- No cloud, account, sharing, project-history, or asset-library feature work.
- No large new effect batch.
- No overloading Merge with mask behavior; Mask remains a graph-only utility.
- No change to existing Repeat default output behavior.
- No user-facing agent workflow or release-planning copy in app routes.
- No non-serializable renderer, DOM, WebGL, Pixi, or canvas objects in
  `CanvasDocument`.

## Acceptance Criteria

- Existing documents normalize with `maskNodes: []`, `transformNodes: []`, and
  fixed Repeat rotation defaults.
- Existing Repeat fixtures stay deterministic and preserve default output.
- A Mask node can be added, connected through `source` and `mask` ports, edited
  in the inspector, rendered in thumbnails, and used as an export upstream node.
- A Transform node can be added after any branch, connected through `source`,
  edited in the inspector, rendered in thumbnails, and used to move, scale,
  rotate, or fade the completed upstream result.
- Alpha, luma, and threshold mask modes produce transparent output where the
  matte excludes pixels.
- Line Field source layers render nonblank output across supported aspect
  ratios and serialize safely.
- Line Field can be used as a layer-backed source node in graph workflows,
  masked, repeated, merged, thumbnailed, and exported.
- Repeat `step` and `random` rotation modes change copy angles while staying
  stable for the same document seed and node seed offset.
- New starter recipes open as graph-backed documents and render nonblank output.
- Docs and roadmap describe the release scope without exposing implementation
  planning language in product surfaces.
