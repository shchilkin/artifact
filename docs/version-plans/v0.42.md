# v0.42 UI Foundation And Cross-Product Proof

Status: release candidate. Delivery is tracked in GitHub milestone
[`v0.42 UI Foundation And Cross-Product Proof`](https://github.com/shchilkin/artifact/milestone/4).

## Purpose

v0.42 establishes one source-owned UI Foundation that Artifact Web and
Backoffice consume through distinct Product Themes. It proves the contract in
the existing Artifact AI Generation composer and Backoffice sign-in before
broad migration begins.

The release thesis is: Artifact and Backoffice can share accessible interaction
mechanics and semantic theming without sharing one visual theme.

## Scope

- Add the shared UI workspace package and semantic Theme Contract.
- Provide Artifact and Backoffice Product Themes with a shared Brand Signature.
- Ship the accepted v1 primitive catalog: Button, ButtonLink, IconButton, Field,
  Input, Textarea, NativeSelect, InlineNotice, Skeleton, ProgressIndicator,
  Tooltip, and Popover.
- Show the same Foundation Matrix in Artifact `/docs/style-guide` and
  Backoffice `/style-guide`.
- Migrate the existing Artifact AI Generation composer and Backoffice sign-in
  as the two proof consumers.

## Non-Goals

- No broad Artifact or Backoffice migration.
- No universal cross-product visual theme.
- No new AI Chat capability or product behavior.
- No renderer, graph, document, persistence, API, auth, or business-rule change.

## Acceptance Criteria

- Both applications consume one source-owned primitive API.
- Both Product Themes satisfy the Theme Contract and remain visually distinct.
- Both Foundation Matrix routes expose the same primitive/state identifiers.
- The Artifact AI Generation composer and Backoffice sign-in preserve their
  existing behavior.
- Focus, keyboard, disabled, error, loading, responsive, and theme-isolation
  states have focused automated coverage.
