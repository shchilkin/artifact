# v0.46.0 Release Notes

Artifact v0.46.0 is the Inspector System release. It gives Layers, Nodes,
effects, materials, scenes, and shader authoring one precise, accessible
property-editing language while preserving the creative behavior underneath.

This release rewrites the presentation and composition of property inspectors.
It deliberately leaves node-canvas, preview, thumbnail, gallery, and 3D chrome
to v0.47 and final legacy removal to v0.48.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, persistence, AI scope, provider behavior, package export, and font
policy remain unchanged.

## Highlights

- Unified ordinary and dense inspector sections, fields, property rows,
  readouts, notices, validation, status, and disclosure through source-owned
  Artifact patterns.
- Migrated layer and common graph properties while preserving authoritative
  locked, hidden, connected, and editing-target state.
- Migrated effect, material, and scene controls, including an explicit
  read-only distinction for graph-connected material inputs.
- Migrated Code Shader and AI Shader authoring with associated labels, accepted,
  dirty, invalid, loading, and failure states.

## Scope Boundaries

- Included: layer, graph-property, effect, material, scene, Code Shader, and AI
  Shader inspector composition and accessible state presentation.
- Included: deterministic live style-guide specimens plus focused desktop,
  narrow, keyboard, runtime, and cross-browser coverage.
- Deferred: node-canvas, port, preview, thumbnail, gallery, and 3D viewport
  chrome to v0.47.
- Deferred: compatibility-alias removal and final cross-app conformance to
  v0.48; new AI-assisted creation remains v0.49.

## Validation

Local validation passed during release prep on 2026-07-24:

- `npm run release:verify -- --version 0.46.0`
- `npm run check`
- `npm run build`
- `PLAYWRIGHT_WEB_SERVER_PORT=4174 npm run test:browser:release`
- focused Artifact style-guide, live Layers/Nodes, Code Shader, AI Shader,
  material, scene, narrow-drawer, Chromium, and Firefox checks

The full browser gate passed with `501 passed, 51 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, mobile WebKit, and Backoffice
desktop/mobile Chromium.

Performance notes:

- `npm run perf:node-editor` was not required because v0.46 changes no graph
  traversal, renderer algorithm or signature, thumbnail queue, or canvas
  interaction hot path.

## Manual QA

- Reviewed the live ordinary and dense inspector specimens and representative
  Layers, Nodes, material, scene, Code Shader, and AI Shader surfaces.
- Verified keyboard disclosure, associated status and validation text, and
  narrow-drawer containment, including font actions inside a 340-pixel property
  drawer.

## Accepted Risks

- Conformance covers representative live surfaces, semantics, keyboard focus,
  native states, geometry, responsive overflow, and the supported browser
  matrix; it is not a complete visual-regression suite or WCAG certification.
- Compatibility wrappers and legacy aliases remain until the v0.48 zero-caller
  conformance gate.
- The build retains inherited Fast Refresh warnings and the large Three.js
  vendor-chunk warning. Browser logs retain existing blank-frame fallback and
  React Flow development warnings; no release-gate scenario failed.
