# v0.36.0 Release Notes

Artifact v0.36.0 is the 3D Model, Scene Lighting, And Retro Finishes release.
It lets imported 3D assets become editable graph ingredients, then pushes them
through scene lighting, environment maps, and old-game image effects.

This release changes 3D and retro-effect workflows without turning into a full
3D editor, material authoring package, cloud asset pipeline, or broad palette
library release.

Document schema, renderer/export behavior, thumbnail scheduling, and browser
test configuration changed to support 3D model and environment graph workflows.
Graph traversal, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added 3D Model and Environment nodes, plus a 3D Scene layer/node that renders
  imported models with camera, lighting, environment, backdrop, and material
  controls.
- Added real Three.js model rendering for GLB/GLTF assets, scene-owned framing,
  undoable viewport edits, live scene thumbnails, export parity, and an
  always-spinning 3D Model input-node preview that does not commit graph state.
- Added retro finishing effects for indexed palettes, dot grain, alpha crush,
  silhouette crush, and edge crush so rendered 3D sources can move toward a
  PlayStation-era poster look.
- Updated Layers vs Nodes semantics so Layers treats 3D Scene as the editable
  layer while 3D Model and Environment appear there as scene inputs; Nodes still
  exposes each ingredient as a graph node.

## Scope Boundaries

- Included: serializable 3D model/environment layer data, graph render
  signatures, model/environment thumbnail rendering, fallback handling for
  unsupported model nodes, and browser asset import for supported 3D/HDR image
  inputs.
- Included: scene camera, model transform, lighting, environment intensity and
  rotation, backdrop input, and export/preview parity for the 3D scene path.
- Included: v0.36 browser coverage for model drop/import, scene render/export,
  layer/node semantics, auto-spinning model previews, unsupported model drops,
  and undoable scene viewport edits.
- Deferred: full OBJ/material-library workflows, per-material editing,
  animation/timeline controls, arbitrary 3D scene composition, named retro
  console palettes, and deeper dither families.
- Deferred: broad WebGL parity coverage in Firefox/WebKit; those browsers keep
  lightweight unsupported-drop coverage while the heavier 3D rendering tests run
  in Chromium.

## Validation

Local validation passed during release prep on 2026-06-16:

- `npm run check`
- `npm run build`
- `npm run test:browser:release`
- `npm run test:browser:chromium -- tests/browser/generator.spec.ts --grep "AI image node appends history"`
- `npm run test:browser:chromium -- tests/browser/v036-3d-model-retro.spec.ts`
- `npm run perf:node-editor`
- `npm run release:verify`

The full browser gate passed with `296 passed, 35 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, and mobile WebKit. A second full browser run
exposed unrelated existing browser-suite flakes; a third full browser run then
passed with the same `296 passed, 35 skipped` result.

Performance notes:

- `npm run perf:node-editor` passed during release prep. Dragging, slider
  changes, and graph panning stayed around 16-17ms frame p95 with zero long
  tasks during interactions. Initial node-editor load still has startup long
  tasks from thumbnail/render/GPU work and remains a follow-up.

## Manual QA

- Imported a skull GLB into a 3D Model node, connected it through 3D Scene, and
  verified scene camera, rotation, positioning, environment, backdrop, merge,
  and export workflows.
- Verified Layers presents 3D Scene as the layer target while model and
  environment inputs remain scene settings there, with explicit graph nodes
  preserved in Nodes.
- Verified retro stacks using indexed palette, dot grain, alpha/silhouette
  crush, edge crush, merge, fill, transform, and export.
- Verified preview/export parity for the retro 3D composition and that 3D Model
  input previews can rotate continuously without polluting undo/history.

## Accepted Risks

- The first supported model path is GLB/GLTF. OBJ-style workflows remain future
  polish because they need separate material/texture handling to feel reliable.
- `three-vendor` is now a large production chunk. This is acceptable for the
  first 3D release, but deeper code splitting and lazy 3D loading remain
  performance follow-ups.
- Initial node-editor load still has startup long tasks in the benchmark. The
  interactive drag, slider, and pan scenarios stay within the preferred frame
  budget.
- WebKit can emit benign ResizeObserver and Vite dynamic-import noise during
  local browser tests; the test harness filters those known non-product
  messages while still failing on product errors.
