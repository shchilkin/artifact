# v0.39.0 Release Notes

Artifact v0.39.0 is the Account Cloud Saves And Production Auth release. It
adds a self-owned account path, cloud project saves, password recovery, and
production API hardening while preserving Artifact's local-first editor model.

This release changes authentication, account-backed project persistence, cloud
asset sync, project sync status UI, and production deploy behavior. It
deliberately does not add public share links, collaboration, team ownership,
project history/versioning, S3 object storage, renderer semantics, or graph
traversal changes.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Replaced Clerk with Better Auth so Artifact owns the sign-up, sign-in,
  session, and account-backed project flow through the API.
- Added password recovery with Resend-backed reset emails and a dedicated reset
  password route.
- Added cloud project saves that keep local projects and recovery drafts intact
  while allowing signed-in users to upload and load cloud copies.
- Split large cloud projects into a compact project manifest plus separately
  uploaded assets so image/font/model/environment payloads do not have to live
  inside one oversized project request.
- Hardened production deploys with startup migrations, Coolify healthchecks,
  Vercel preview API origin support, and Redis persistence fixes.
- Clarified Projects sync UI with explicit local, cloud, synced, syncing,
  failed, and too-large states.
- Continued editor polish with typography-system improvements, selected editor
  UI refinements, and a node-canvas menu drag lifecycle fix.

## Scope Boundaries

- Included: Better Auth account flow, password recovery, cloud project APIs,
  cloud project asset APIs, client project reassembly, Projects sync status UI,
  production migrations, Coolify healthchecks, Redis persistence configuration,
  and Vercel preview API origins.
- Included: focused editor typography/UI polish and the node canvas menu drag
  lifecycle fix that landed after v0.38.
- Deferred: public share links, permissioned sharing, ownership transfer,
  collaboration, account settings, quota/usage UI, and team/workspace features.
- Deferred: S3-compatible object storage, cross-project asset deduplication,
  storage cleanup policy, project autosave history, project versions,
  restore/compare, renderer/export semantic changes, graph traversal changes,
  and broad public-site identity work.

## Validation

- `npm run release:verify`
- `npm run check`
- `npm run build`
- `npm run test:browser`

The full browser gate passed with `340 passed, 38 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, and mobile WebKit.

Performance notes:

- `npm run perf:node-editor` was not required because v0.39 changes
  account-backed persistence, API deploy behavior, project asset sync, and
  Projects/status UI rather than React Flow gesture paths, graph traversal,
  thumbnail scheduling, renderer/export behavior, or performance-sensitive
  node-editor interactions.

## Manual QA

- Verified account creation, sign-in, sign-out, password reset email delivery,
  and reset-password completion during the production auth debugging pass.
- Verified local project to cloud project behavior, including a large
  asset-backed project that previously showed "Project is too large to sync."
- Verified Projects UI communicates local, synced, syncing, failed, and
  too-large sync states with clear retry/attention affordances.
- Verified production API logs and Coolify health after migration and
  healthcheck changes.

## Accepted Risks

- Public share links, sharing permissions, ownership transfer, and team/project
  collaboration remain future work. Cloud projects are private account saves in
  this release.
- Cloud assets remain on the initial server storage path rather than
  S3-compatible object storage. S3 migration, quotas, cleanup, and asset
  deduplication remain future cloud-storage work.
- Project history, autosave versions, restore/compare, and explicit
  cloud/local conflict resolution remain deferred so the first account-backed
  save path can ship with a smaller behavioral surface.
