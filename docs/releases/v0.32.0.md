# v0.32.0 Release Notes

Artifact v0.32.0 is the Code Health and Debt Reduction release. It turns the
post-v0.31 Fallow cleanup into a safer maintenance baseline by reducing
duplication and full-health complexity findings to zero, cleaning React hook
warnings, documenting the CSS/Tailwind boundary, and preparing the next debt
work without changing editor semantics.

This release changes release planning, code-health documentation, package
metadata, hook dependency boundaries, route/controller structure, API helper
structure, and editor helper organization. It deliberately does not ship a new
product feature or broaden renderer/editor behavior.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added the v0.32 code-health plan covering Fallow complexity policy,
  route-level editor orchestration debt, React hook warnings, canvas gesture
  boundaries, CSS/Tailwind strategy, roadmap cleanup, and storage/render risk
  triage.
- Captured the v0.32 Fallow health review in
  `docs/fallow-v0.32-health.md`: health score `91.4`, zero functions above
  threshold, zero critical/high/moderate complexity findings, and changed-code
  audit `verdict: "pass"`.
- Reduced historical complexity hotspots without Fallow suppressions or CI
  threshold weakening by splitting large editor, node-canvas, API, auth, home,
  showcase, random preset, and preview helpers into smaller focused units.
- Fixed React hook dependency warnings in AI generation, node context menus,
  node selection sync, thumbnail rendering, and document rendering without
  weakening lint rules.
- Documented why the node editor remains CSS-first while Tailwind stays useful
  for route shells, wrappers, and simple low-state layout.
- Bumped package metadata to `0.32.0` in `package.json`,
  `apps/web/package.json`, and `package-lock.json`.

## Scope Boundaries

- Included: v0.32 package metadata bump.
- Included: Fallow health cleanup, changed-code audit validation, and full
  complexity report cleanup to zero functions above threshold.
- Included: React hook warning cleanup with stable refs/memoized option
  boundaries where needed.
- Included: focused helper/component extraction across historical complexity
  hotspots where behavior could be preserved with existing or focused tests.
- Included: roadmap, version-plan, production-readiness, and release-notes
  cleanup for the v0.32 release thesis.
- Included: storage/render and CSS/Tailwind risk classification.
- Deferred: converting full-health Fallow complexity into a permanent release
  gate for every future branch until the policy is documented in CI terms.
- Deferred: broad renderer, persistence schema, graph traversal, export, and
  CSS monolith rewrites.
- Deferred: landing work, Showcase / How-to work, command palette,
  server-backed sharing, dynamic OG generation, graph traversal changes, export
  changes, persistence schema changes, AI scope changes, package export
  changes, and font-policy changes.

## Accepted Risks

- Full-health complexity is clean for this release, but making that report a
  permanent strict release gate for every future change still needs a documented
  CI policy.
- Broad renderer, persistence, graph traversal, export, and CSS monolith
  rewrites are deferred until each has focused tests and a narrower release
  thesis.
- npm install previously reported the existing dependency audit state: 11
  vulnerabilities (4 moderate, 7 high). npm audit remediation is outside the
  v0.32 code-health release scope and is not part of the current release gate.
