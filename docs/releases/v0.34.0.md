# v0.34.0 Release Notes

Artifact v0.34.0 is the Active Project Save Model release. It makes local
Projects behave like editable projects instead of one-off snapshots, so saving
an already-loaded project updates that same saved item.

This release changes the local Projects workflow, active project state,
Projects panel copy, and the local Projects page. It deliberately does not add
account sync, cloud project storage, server-backed sharing, broad asset
library, export history, autosave history, or project version history.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, AI scope, package export, and font policy remain unchanged.

## Highlights

- Added an editor-session active project binding outside `CanvasDocument`, so
  loading or creating a project lets `Save Project` update that same local
  project record.
- Changed first-save, overwrite, rename, reload, and contextual copy behavior so
  Projects now has a clear primary save path and a separate branch/copy action.
- Reworked Projects status copy around `Not saved as project`, `Unsaved
  changes`, and `Saved in project`, with storage diagnostics still kept behind
  `Storage details`.
- Added `/projects` as a first-class local workspace page for browsing saved
  projects and recovery copies before opening them in the editor.
- Disabled the primary save action when the active project is already clean, so
  the UI no longer invites redundant saves.
- Added unit and browser coverage for active project binding, project overwrite
  behavior, dirty-state transitions, copy behavior, Projects panel layout, and
  the dedicated Projects page.
- Added machine-checked release metadata and a manual production release
  workflow so future tags and GitHub Releases are verified from the same
  template-backed release notes.

## Scope Boundaries

- Included: v0.34 package metadata bump.
- Included: active project id binding stored outside exported document data.
- Included: `Save Project` overwrite semantics for the active local project.
- Included: contextual `Copy` behavior for creating a separate active project
  branch.
- Included: Projects panel wording and layout polish for the new save model.
- Included: `/projects` route for browsing local saved projects and recovery
  copies outside the editor sheet.
- Included: release metadata verification and a manual release workflow for the
  future `main` production branch.
- Deferred: project autosave history, named versions, restore/compare UI,
  server-backed share records, account sync, and cloud storage.
- Deferred: renderer output, graph traversal, document schema, package export,
  AI scope, and font-policy semantic changes.

  the editor with the active project binding preserved.

## Accepted Risks

- Active project binding is local browser state only. Exported documents,
  share payloads, and project packages do not carry the binding.
- Recovery copies remain independent from saved projects. A future autosave
  history surface should separate explicit saves, automatic recovery points, and
  user-created copies more clearly.
- `/projects` is a local browser library, not an editor replacement. Explicit
  saving, active project overwrite, and copy/branching remain inside `/app`;
  richer autosave history, restore, compare, and version workflows remain
  future work.
