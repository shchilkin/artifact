# v0.41.1 Release Notes

Artifact v0.41.1 is the 3D project package portability patch. It fixes a
problem discovered while validating v0.41.0: exported `.artifact` project
packages could preserve 3D graph structure and asset references without
including the imported model and environment payloads needed to reopen the
project elsewhere.

This release changes project package preparation and local 3D asset recovery.
It does not change graph traversal, scene rendering semantics, or the document
schema.

Document schema, graph traversal, renderer behavior, thumbnail scheduling, AI
scope, and font policy remain unchanged. Package export now embeds imported
GLB/GLTF models and HDR/EXR environments alongside existing portable assets.

## Highlights

- `.artifact` packages now include imported GLB/GLTF and HDR/EXR payloads with
  their stable reference, MIME type, name, and size metadata.
- Importing a package into a clean browser restores those payloads to local
  asset storage before the document opens, so model and environment references
  resolve without selecting the files again.
- Version 1 packages and schema version 3 documents remain supported. Older
  packages that never contained a 3D payload continue to open safely and expose
  the unresolved dependency instead of manufacturing replacement data.

## Scope Boundaries

- Included: imported local models referenced by `artifact-model://...`.
- Included: imported local HDR/EXR environments referenced by
  `artifact-env://...`.
- Deferred: recovery of binary data that was already omitted from an older
  exported package; the original file must still be available to repair it.
- Deferred: external glTF sidecars and cloud-hosted asset synchronization.

## Validation

Release validation completed on 2026-07-19:

- `npm run release:verify`
- `npm run check`
- `npm run build`
- `npm run test:browser:release`
- package serialization/import unit tests for embedded GLB and EXR payloads
- Chromium export, new browser context, import, IndexedDB resolution, and
  rendered 3D Scene regression coverage
- a manual round trip with a 2.9 MB GLB and 6.1 MB EXR in a clean browser
  context

The full browser gate passed with `354 passed, 47 skipped` across Chromium,
Firefox, WebKit, mobile Chromium, and mobile WebKit. The release candidate also
passed the development CI matrix and exact-SHA staging revision checks.

Performance notes:

- `npm run perf:node-editor` was not required because the patch does not change
  node-editor interaction, graph traversal, or rendering performance paths.

## Manual QA

- Exported a project whose model and environment existed only in IndexedDB,
  then imported it into a new browser context with empty storage.
- Confirmed exact model and environment byte sizes, stable references, graph
  connections, scene settings, and a non-fallback rendered 3D Scene after the
  import.

## Accepted Risks

- A package exported before this fix cannot recover GLB or EXR bytes that were
  never embedded; Artifact reports those dependencies as missing.
