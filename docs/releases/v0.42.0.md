# v0.42.0 Release Notes

Artifact v0.42.0 is the UI Foundation And Cross-Product Proof release. It adds
one source-owned interaction foundation that Artifact Web and Backoffice consume
through distinct Product Themes, then proves that contract in the existing
Artifact AI Generation composer and Backoffice sign-in.

This release establishes the shared UI boundary and the first two consumers. It
deliberately does not complete the broad UI rewrite or add new AI Chat product
capability.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, persistence, AI scope, package export, and font policy remain
unchanged.

## Highlights

- Added the internal `@artifact/ui` workspace with the source-owned Button,
  ButtonLink, IconButton, Field, Input, Textarea, NativeSelect, InlineNotice,
  Skeleton, ProgressIndicator, Tooltip, and Popover primitives.
- Added one semantic Theme Contract, distinct Artifact and Backoffice Product
  Themes, and a shared Artifact Brand Signature without imposing one visual
  theme on both products.
- Added the same package-owned Foundation Matrix to Artifact
  `/docs/style-guide` and Backoffice `/style-guide`.
- Migrated the existing Artifact AI Generation composer and Backoffice sign-in
  as the two proof consumers, including keyboard, pending, error, responsive,
  and signed-out states.
- Added cross-product package-boundary and browser release gates, including
  Backoffice UI coverage and WebKit-native control geometry.

## Scope Boundaries

- Included: the shared command, field, feedback, async-state, Tooltip, and
  Popover mechanics plus both Product Themes.
- Included: two live Foundation Matrix routes, the Artifact AI Generation
  composer, Backoffice sign-in, and the cross-product conformance gate.
- Deferred: broad Backoffice, Artifact product-surface, editor workflow,
  inspector, and canvas-chrome migration across v0.43 through v0.47.
- Deferred: compatibility-wrapper and legacy-selector removal until the v0.48
  conformance gate, followed by new AI-Assisted Creation work in v0.49.

## Validation

Local validation was rerun against the clean `origin/development` base during
release prep on 2026-07-22:

- `npm run release:verify -- --version 0.42.0 --skip-tag-check`
- `npm run check`
- `npm run build`
- `npm run test:browser:release`
- focused Chromium AI Generation composer/failure verification
- focused WebKit Foundation Matrix verification

`npm run check` passed formatting, lint, type checking, 25 deployment tests,
20 UI Foundation tests, 677 Web tests, 261 passing API tests with 5 skipped,
and 9 Backoffice tests. Both production builds completed successfully.

The full browser gate completed with 406 passed and 49 intentionally skipped
tests across
Artifact Chromium, Firefox, WebKit, mobile Chromium, mobile WebKit, and
Backoffice desktop/mobile Chromium. One Chromium GLB/EXR clean-context test
used its available retry because the test helper still opened hard-coded port
`4173` while the isolated gate ran on `4174`. After binding the helper to
Playwright's configured `baseURL`, the scenario passed 1/1 with retries
disabled.

Performance notes:

- `npm run perf:node-editor` was not required because v0.42 does not change
  graph traversal, renderer behavior, thumbnail scheduling, or node-canvas hot
  paths.

## Manual QA

- Reviewed both live Foundation Matrix surfaces at desktop and mobile widths;
  they share anatomy and state coverage while retaining visibly distinct
  Product Themes and the common Artifact Brand Signature.
- Reviewed the AI Generation composer field, command, pending, progress, and
  provider-error states after its migration to Foundation controls.
- Reviewed Backoffice sign-in default, pending, error, autofill, and keyboard
  states after its migration to the Foundation controls.

## Accepted Risks

- Most Artifact and Backoffice surfaces still use local or legacy CSS. Their
  bounded migration remains explicit v0.43-v0.48 work rather than hidden v0.42
  scope.
- Artifact compatibility wrappers and compatible selectors remain until the
  v0.48 zero-caller gate proves they can be removed safely.
- The conformance gate covers semantics, keyboard focus, native states,
  geometry, overflow, theme isolation, and limited style fingerprints; it is
  not a full visual-regression suite or a complete WCAG certification.
- `@artifact/ui` is an internal source-consumed workspace, not a published
  public package.
- The build retains 11 existing Fast Refresh warnings and the existing large
  Three.js vendor-chunk warning; neither is introduced by the UI Foundation.
