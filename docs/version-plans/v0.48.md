# v0.48 UI Conformance And Legacy Removal

Status: planned. Blocked by the v0.43 through v0.47 release gates. Delivery is
tracked in GitHub milestone [`v0.48 UI Conformance And Legacy Removal`](https://github.com/shchilkin/artifact/milestone/5).

## Purpose

v0.48 closes the full UI rewrite by enforcing conformance across both products
and removing only legacy contracts with proven replacements and zero callers.

The release thesis is: Artifact and Backoffice have complete, enforceable
UI-system coverage with no unapproved legacy bucket.

## Scope

- Expand the migration inventory into an enforceable legacy registry.
- Migrate residual Artifact and Backoffice callers.
- Establish two-product visual and accessibility conformance.
- Contract deprecated primitives, aliases, exports, selectors, and CSS.
- Keep AI-Assisted Creation paused until the final gate closes.

## Non-Goals

- No new product feature, design direction, or component-catalog expansion.
- No deletion without a proven replacement and zero registered callers.
- No renderer, graph, document, persistence, API, auth, or AI behavior change.

## Acceptance Criteria

- The inventory and legacy registry contain no unresolved, unapproved entry.
- Both Foundation Matrix routes expose the same v1 primitive/state identifiers
  under their respective themes.
- Representative product flows pass visual, responsive, keyboard, focus,
  reduced-motion, and accessibility conformance.
- Removed contracts cannot be reintroduced without failing an automated guard.
- Closing Q6 unblocks v0.49 but does not implement AI scope.
