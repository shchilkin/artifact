# v0.23.0 Release Notes

Artifact v0.23 is the Graph Add Library Drag release. It makes node-graph
construction more direct by letting Add Library items land on existing graph
edges, while keeping the shared catalog, document schema, graph traversal,
renderer/export behavior, and thumbnail scheduling unchanged.

## Highlights

- Add Library rows in the node menu can now be dragged onto an existing edge to
  insert a new node into that connection.
- Edge insertion uses a pure graph insertion helper to resolve the target edge
  and existing document commands to split the edge through the new node.
- Empty-canvas drag-to-add, click-to-add, right-click add, and dropped-handle
  add flows remain intact.
- Dragging over a valid edge adds a readable edge insertion affordance without
  changing the graph until drop.
- Inserted layer-backed nodes preserve layer ordering around the replaced edge,
  and graph-only utility nodes stay graph-only.
- Undo/redo treats one drag-to-edge insertion as one document update.
- Browser coverage now includes drag-to-edge insertion and undo/redo around the
  split edge.

## Validation

Local validation passed on 2026-05-26:

- `npm run check`
- `npm run build`
- `npm run test:browser` across Chromium, Firefox, WebKit, mobile Chromium, and
  mobile WebKit. The full browser gate reported `206 passed, 11 skipped`.
- `npm run perf:node-editor`
- Focused Chromium browser coverage verified Add Library drag-to-canvas and
  drag-to-edge insertion.
- Unit coverage verifies insertion-target selection and graph edge split
  behavior.

Performance notes:

- Interaction scenarios for visible node drag, effect slider changes, and large
  graph pan had `0` long tasks with p95 frame times around `17ms`.
- Initial node load still reports expected first-load long tasks from render
  and GPU setup work; this release does not expand thumbnail scheduling or
  renderer scope.

Manual validation:

- Manual user QA is still recommended before tagging if the release branch is
  not merged immediately after this prep.

## Accepted Risks

- Add Library drag insertion currently targets edges and empty canvas. Dropping
  onto specific handles and multi-node recipe drag remain future candidates.
- Edge hit testing uses the same graph-space node centers as existing node
  intercept behavior; very dense graphs may still need future hover refinement.
- Playwright WebKit is the automated Safari-family gate. A manual macOS Safari
  pass is still recommended before a broad public announcement.
