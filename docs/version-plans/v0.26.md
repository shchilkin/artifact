# v0.26 Layer Mode Polish

## Summary

v0.26 makes Layers feel first-class again after several releases that strengthened
Nodes, typography, packaging, and the shared Add Library.

The goal is not to turn Layers into a simplified node graph. Layers should stay
linear, direct, and fast: add something, see the final result, adjust it, keep
moving. The editor should communicate that mental model clearly while preserving
the shared document, renderer, export, thumbnail, and graph boundaries.

This is a product/editor polish slice. It should not change `CanvasDocument`
schema, graph traversal, render/export semantics, thumbnail scheduling, AI
scope, or package/font policy.

## Goals

- Make the Layers workflow feel as deliberate and confident as Nodes.
- Keep the Add Library shared, but tune the Layers presentation for fast linear
  composition.
- Improve empty and near-empty layer starts so users can begin without switching
  contexts.
- Make selected layer state, visibility, muted/hidden state, and reorder targets
  easier to read in a dark low-chrome UI.
- Improve layer-edit feedback for common operations: select, rename, duplicate,
  hide/show, reorder, add effect/source, and adjust controls.
- Preserve quick preview responsiveness for stack/layer work.
- Keep Layers and Nodes behavior parity where parity matters: same library
  items, same effect controls, same export result, same document model.

## Non-Goals

- No new durable document fields.
- No new graph node kinds.
- No renderer algorithm changes unless a focused test exposes an existing
  preview/export mismatch.
- No server-backed projects, account sync, or share-link storage.
- No new font import, font catalog, or package export behavior.
- No corporate/admin redesign, neutral dashboard treatment, or verbose setup
  flow.

## Product Principles

- **Layers are linear.** They should answer "what is my final image right now?"
  faster than Nodes answers "how is this graph constructed?"
- **Nodes are structural.** Do not copy graph-only affordances into Layers unless
  they make linear editing clearer.
- **Shared library, surface-specific behavior.** Layers and Nodes should use the
  same Add Library model, but Layers can prioritize click-to-add and compact
  hover previews while Nodes can prioritize drag-to-canvas and edge insertion.
- **State must be visible without shouting.** Use the existing warm dark visual
  system: tonal separation, 1px rules, deliberate accent, and clear focus/active
  states.
- **No render surprises.** Every layer-mode change must preserve preview/export
  parity and the existing stack-vs-graph boundary.

- [ ] Walk the current Layers path from blank document to exported image.
- [ ] Capture friction points for empty state, add menu, selection, controls,
  reorder, visibility, and export confidence.
- [ ] Compare with Nodes only where the shared system should behave the same.
- [ ] Record any behavior that feels slow separately from visual clarity issues.

### 2. Layer Add Library Polish

- [x] Keep the shared Add Library data/model introduced in v0.17-v0.18.
- [x] Make the Layers Add Library use the richer hover-preview treatment already
  proven in Nodes, tuned for the sidebar width and layer-first workflow.
- [x] Keep click-to-add primary in Layers.
- [ ] Keep drag optional in Layers only if it has a clear insertion/reorder
  target and does not conflict with layer row drag behavior.
- [x] Ensure source items such as Fill, Image, Text, and AI Image have previews
  that represent the item itself, not a generic effect sample.
- [x] Preserve search, categories, popular/recent/favorite behavior.

Initial implementation note: Layers now exposes AI Image plus noise/array source
presets through the shared Add Library. AI Image remains a layer-mode generation
entry point instead of a new document field, while noise/array presets add normal
source layers.

## Acceptance Criteria

- Layers feel like the fastest path to a final image, not a secondary list view.
- The Layers Add Library is easier to browse than the old long dropdown while
  staying consistent with Nodes.
- Layer row state is readable in the dark editor at normal working zoom.
- Empty and starter states guide action without adding corporate onboarding
  chrome.
- Shared controls remain shared; Layers and Nodes do not fork effect metadata.
- Preview/export output is unchanged for equivalent documents.
- No document schema migration is required.
- No graph traversal, render/export semantics, thumbnail scheduling, AI scope,
  or package/font policy changes are introduced.

## Deferred Candidates

- Server-backed share links.
- Account-synced project and font libraries.
- Google Fonts catalog search.
- Command palette entry for font/library actions.
- Full visual regression snapshot suite.
