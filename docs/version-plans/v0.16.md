# v0.16 Acceptance Plan

## Theme

Editor Workflow Polish.

The goal of v0.16 is to make daily editing feel more obvious and confident
without changing document semantics, renderer output, graph traversal, export
behavior, AI alpha scope, or local project persistence.

v0.15 made the editor easier to read and gave the release a real browser
matrix. v0.16 should use that baseline to polish the workflow: clearer active
states, better layers ergonomics, quieter diagnostics, and a more predictable
release process.

## Product Scope

- Improve editor clarity beyond raw contrast: selected, hovered, focused,
  disabled, draggable, and active states should be easy to distinguish in both
  layers and nodes.
- Polish layer-first editing as a linear workflow: users should understand the
  current stack, selected layer, drag target, visibility state, and folder/area
  state without hunting.
- Add an optional AI diagnostics surface for developer/debug use while keeping
  normal console output quiet.
- Harden release and CI hygiene so warnings and browser-specific caveats are
  documented instead of discovered during release.
- Add focused browser regressions only for flows that unit tests cannot prove.

## Non-Goals

- Do not add new renderer behavior or a second preview/export path.
- Do not change graph traversal, graph area render semantics, node thumbnail
  scheduling, primitive camera semantics, or export behavior.
- Do not add account-backed project sync, server-backed sharing, billing, or
  broad AI product scope.
- Do not redesign Artifact into a generic light UI. The app stays dark, raw,
  mono, and zine-like.
- Do not split CSS purely for tidiness unless the touched UI needs it.

## Acceptance Criteria

- `npm run check`, `npm run build`, and `npm run test:browser` pass before
  release.
- GitHub Actions no longer reports the Node.js 20 JavaScript action runtime
  deprecation warning in the normal release gate.
- Layer and node editor states are visibly distinct: selected, focused,
  hovered, draggable, disabled, hidden, active tab, active toolbar action, and
  drop target.
- Layer editing remains responsive and render output remains unchanged.
- AI diagnostics are opt-in and discoverable for developers, with normal user
  sessions keeping console output quiet.
- Browser coverage still spans Chromium, Firefox, WebKit, mobile Chromium, and
  mobile WebKit. Any browser-specific limitation is documented in the version
  plan and release notes.

## Manual QA

- Open a stack document in layers mode and confirm selected, hovered, hidden,
  dragging, and focused rows are distinguishable.
- Reorder layers and confirm the drop target is obvious.
- Use area/folder rows and confirm collapsed, expanded, selected, renamed, and
  ungroup actions remain discoverable.
- Switch to nodes and confirm selected nodes, selected graph areas, handles,
  toolbar actions, and debug/perf controls read as separate states.
- Enable AI diagnostics and confirm auth/API/debug information appears only
  when explicitly requested.
- Export stack and graph documents and compare preview/export composition.
