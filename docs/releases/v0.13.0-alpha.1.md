# v0.13.0-alpha.1 Release Notes

Artifact v0.13 introduces the private AI image generation alpha. This release
keeps the editor usable without sign-in, but gates AI generation behind an
authenticated, AI-enabled account so provider spend stays bounded.

## Highlights

- Added the AI Image node workflow as a normal image layer with account-gated
  generation controls.
- Added Clerk-backed browser sign-in and backend user provisioning with AI
  disabled by default.
- Added the VPS API, worker, BullMQ queue, Postgres schema, generated asset
  storage, Bull Board support, and provider adapters for OpenAI and xAI.
- Added monthly quota, one-active-job guard, create rate limiting, and
  idempotency handling on the backend.
- Added generated image history, previous/next traversal, retry/recovery
  actions, compact job diagnostics, and failed-state overlays.
- Added cleanup operations for stale generated jobs, orphan generated asset
  records, and orphan local generated files.

## Accepted Risks

- This is a private alpha. AI access remains restricted to explicitly enabled
  users.
- Generated outputs are imported into browser-local `artifact-asset://...`
  storage. Server-side project sync is still out of scope.
- Failed provider requests may consume quota once the provider call has started.
  Idempotent duplicate requests do not consume extra quota.
- xAI provider exposure and default settings are still subject to provider
  research.
- Cleanup is manual and dry-run-first. It should be run by an operator after
  checking the intended `DATABASE_URL` and `ASSET_STORAGE_DIR`.
- Bull Board is an operator debugging tool and must stay private.
- Coolify/VPS prebuilt-image deploys are planned, but the alpha can merge
  before the pull-only deploy pipeline is finished.
- Production HTTPS for `api.artifact.shchilkin.dev` is temporarily blocked by a
  Let's Encrypt registered-domain rate limit caused by earlier Coolify preview
  certificate churn. The API is healthy behind Traefik, but browser/Vercel
  integration must wait until a trusted certificate is issued.

## Operator Notes

- Grant alpha access with `npm --workspace @artifact/api run grant:ai -- <user-id> [email]`.
- Run cleanup dry runs with `npm --workspace @artifact/api run cleanup:ai`.
- Apply cleanup with `npm --workspace @artifact/api run cleanup:ai -- --apply`.
- Use Bull Board to inspect stuck jobs before applying cleanup.
- If the public API returns `TRAEFIK DEFAULT CERT`, inspect Coolify proxy logs
  for ACME rate limits before changing app code; the backend may still be
  healthy behind the proxy.
