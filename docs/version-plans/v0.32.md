# v0.32 Code Health And Debt Reduction

## Purpose

v0.32 is a code-health release. It should make the editor easier to change by
reducing the highest-value debt that remains after the v0.31 Fallow cleanup:
historical complexity hotspots, route-level orchestration density, React hook
warnings, high-frequency canvas gesture updates, the large node-canvas CSS
surface, stale roadmap structure, and known storage/render risks.

This release is not a product-feature release. Its job is to improve
maintainability, confidence, and change speed without changing editor semantics,
render/export behavior, graph traversal, persistence format, document schema,
AI scope, package export, or font policy unless a small verified bug fix falls
out of the debt work.

Release-prep status on 2026-06-06: implementation is in PR #74 and release
notes are staged in `docs/releases/v0.32.0.md`. Do not tag, publish, or merge
until the final full release gate is green and the maintainer explicitly
confirms the release.

## Product Direction

Artifact now has a stable local-first editor core with layers, nodes, project
storage, export, browser coverage, and Fallow changed-code gates. The next code
quality pass should protect that core before another product slice by answering
practical maintenance questions:

- Which historical complexity hotspots are real refactor targets?
- Which hotspots are intentional public API or workspace plumbing that should
  be documented or suppressed instead of repeatedly rediscovered?
- Which route/controller responsibilities make editor changes harder to review?
- Which React hook warnings are safe to fix, and which need stable callback or
  ref boundaries?
- Which gesture paths still write to document state too often?
- Which CSS belongs in route-level Tailwind utilities, shared primitives, or
  feature CSS?
- Which known storage/render risks deserve near-term action versus a future
  dedicated version?

## Non-Goals

- No landing redesign.
- No Showcase / How-to split.
- No command palette or Add Library feature work.
- No server-backed sharing, account sync, or backend product feature.
- No broad UI rewrite.
- No broad Tailwind migration of node editor chrome.
- No visual restyle of the editor.
- No renderer, graph traversal, export, persistence, document-schema, package
  export, AI scope, or font-policy semantic changes unless a focused debt fix
  exposes and validates a small bug.
- No full-health Fallow complexity gate until the historical baseline is
  reviewed, noisy findings are configured or suppressed, and real refactor
  targets are covered by tests.
- No automatic deletion from Fallow findings without trace validation and the
  normal test gate.

### 2. Route-Level Editor Orchestration Debt

- [x] Keep user-facing copy unchanged unless it is part of a documented docs

### 3. React Hook Warning Cleanup

- [x] Review the current React hook dependency warnings from CI and local lint.
- [x] Fix missing dependencies where the dependency is stable and the behavior
  is clearly preserved.
- [x] Use stable callbacks, refs, memoized options, or narrower helper
  functions where adding a dependency would cause unintended render loops or
  expensive thumbnail/render churn.
- [x] Leave a short code comment only when the stable-boundary choice would be
  non-obvious to a future maintainer.
- [x] Keep lint green without weakening the lint rule or suppressing warnings
  broadly.

### 4. Canvas Gesture Update Boundary

- [x] Audit `CanvasHandles` and related text/image transform gesture paths for
  document writes during pointer moves.
- [x] Move safe text/image transform movement toward the existing rule: draft
  locally during the gesture, commit once when the gesture ends, and invalidate
  only affected render paths. Existing node-image gesture coverage already
  protects the local selected-node path; broader layer-canvas movement is
  deferred because it affects live preview parity and undo semantics.
- [x] Preserve undo/redo semantics, selected layer behavior, thumbnail
  invalidation, save/open behavior, and export parity.
- [x] Add the lowest useful tests before changing any gesture behavior, starting
  with pure state/update helpers and only adding browser coverage for pointer
  interactions that cannot be proven lower.

### 5. CSS Boundary And Tailwind-First Study

- [x] Inventory `apps/web/app/components/node-canvas/node-canvas.css` by
  feature area: canvas shell, nodes, handles, edges, areas, menus, inspector,
  thumbnails, gallery, primitive camera controls, and responsive behavior.
- [x] Decide which styles belong in Tailwind utilities, shared primitives,
  feature CSS, or tokens.
- [x] Treat Tailwind-first as helpful for simple route/page layout, flex/grid
  composition, spacing, alignment, responsive visibility, and low-state
  wrappers.
- [x] Do not use Tailwind-first as the default for node editor chrome,
  graph-specific geometry, canvas/thumbnail aspect behavior, complex selected /
  muted / output-path / drag / hover states, or repeated stateful class strings.
- [x] Prefer feature CSS with semantic tokens for editor and graph surfaces;
  split the monolith by ownership only after the feature boundaries are clear.
- [x] Move repeated control chrome into shared source-owned primitives when the
  same behavior and visual contract appears on multiple surfaces.
- [x] Update the design-system or app-structure docs with the final Tailwind /
  CSS boundary decision if the study changes the current rule.

Tailwind-first assessment:

- Helpful: route shells, one-off spacing around existing components, simple
  responsive layout, alignment wrappers, and small low-state surfaces where
  utility classes stay short and non-branchy.
- Risky: replacing `node-canvas.css` wholesale. The node editor has dense
  graph-specific geometry, pseudo-elements, drag states, selected/output-path
  states, responsive canvas behavior, and product-specific tokens. Moving that
  to long JSX class strings would likely make state harder to scan and reuse.
- Recommended v0.32 approach: keep editor/graph styling CSS-first, use Tailwind
  only at route/wrapper boundaries, and reduce the CSS file by splitting
  feature-owned sections plus migrating repeated controls to shared primitives.

## Acceptance Criteria

- v0.32 has a fresh complexity/debt report and a reviewed classification of the
  top historical hotspots.
- Real complexity refactors are covered by focused tests before behavior is
  changed.
- React hook warnings introduced by current debt targets are fixed or narrowed
  without weakening lint rules.
- Any route/controller extraction preserves document, asset, project, export,
  and node-canvas ownership boundaries.
- Any gesture-boundary fix preserves undo/redo, thumbnail invalidation,
  save/open, preview, and export behavior.
- The `node-canvas.css` debt has an explicit Tailwind/CSS boundary decision,
  and any CSS reduction follows feature ownership instead of a broad utility
  rewrite.
- Stale roadmap planning language is removed or clearly marked historical.
- Storage/render risks are classified, with no more than one or two safe
  improvements pulled into release scope.
- Product feature candidates remain deferred out of v0.32.
