# v0.43.0 Release Notes

Artifact v0.43.0 is the Backoffice UI System release. It moves every Backoffice
route and operational state onto the shared UI Foundation and a coherent set of
Backoffice-owned operational patterns.

This release rewrites the Backoffice interface while preserving account,
authorization, quota, provider-usage, recovery, routing, and API behavior. It
does not migrate Artifact product or editor surfaces.

Document schema, graph traversal, renderer/export behavior, thumbnail
scheduling, persistence, AI scope, package export, and font policy remain
unchanged.

## Highlights

- Migrated the Backoffice shell, navigation, route states, overview, account
  directory, account detail, provider usage, and reconciliation views to the
  shared UI Foundation.
- Added Backoffice-owned operational patterns for page framing, metrics, dense
  data tables, pagination, status, audited controls, and mutation results.
- Preserved keyboard and URL-driven filtering, explicit loading and error
  states, mobile table containment, and deterministic focus after mutation
  results.
- Documented every operational pattern in the live Backoffice `/style-guide`,
  including recovery pending, repeated, success, and failure states.
- Removed the superseded broad Backoffice `Ui.tsx` helper and legacy command and
  field selectors after their callers migrated.

## Scope Boundaries

- Included: every current Backoffice route, operational state, form, data table,
  mutation flow, and the Backoffice live style guide.
- Included: focused desktop/mobile, keyboard, focus, overflow, URL-state,
  conflict, idempotency, and recovery browser coverage.
- Deferred: Artifact public and route-level product surfaces to v0.44.
- Deferred: Artifact editor workflows, inspectors, canvas chrome, and final
  cross-product conformance to v0.45 through v0.48.

## Validation

Local validation passed during release prep on 2026-07-22:

- `npm run release:verify -- --version 0.43.0 --skip-tag-check`
- `npm run check`
- `npm run build`
- `npm run test:browser:backoffice`
- `npm run test:browser:release`

The full browser release gate completed with 432 passed, 49 intentionally
skipped, and one successful retry. The retry was the pre-existing Firefox AI
image reload scenario; it passed on retry and is outside the Backoffice v0.43
change surface. The focused Backoffice matrix completed with 58 passed and 2
viewport-specific skips.

Performance notes:

- `npm run perf:node-editor` was not required because v0.43 changes only the
  Backoffice UI and does not touch graph traversal, rendering, thumbnail
  scheduling, or node-canvas hot paths.

## Manual QA

- Reviewed the Backoffice shell, overview, account directory, account detail,
  provider usage, mutation controls, and operation recovery at desktop and
  mobile widths.
- Reviewed the Backoffice Foundation Matrix and reduced operational specimens,
  including loading, empty, error, pending, repeated, success, and failure
  states.

## Accepted Risks

- Backoffice remains a React Router SPA; an SSR runtime, server-side protected
  loading, and its deployment boundary remain separate infrastructure work.
- Conformance covers semantics, keyboard focus, native states, geometry,
  overflow, responsive behavior, and style fingerprints; it is not a complete
  visual-regression suite or WCAG certification.
- Artifact product and editor surfaces intentionally retain their current UI
  until the bounded v0.44-v0.48 migration releases.
- The build retains the existing Fast Refresh warnings and large Three.js
  vendor-chunk warning; neither is introduced by the Backoffice rewrite.
