# v0.25 Typography Library v3 & Font Policy

## Summary

v0.25 combines the next typography slice with the remaining font-package policy
work. The goal is to make external/open fonts easier to bring into the editor
while keeping editable project export honest about font-file distribution.

This is still a browser-first editor slice. It should not change document render
semantics, graph traversal, thumbnail scheduling, raster export, or AI scope.

## Goals

- Add a Google Fonts import path from the Font Library without requiring a
  Google API key.
- Preserve imported font metadata: source, source URL, license, and embedding
  policy.
- Treat Google Fonts imports as allowlisted open-license fonts that may be
  included in editable project packages with notice metadata.
- Keep unknown local font files metadata-only by default.
- Add an explicit package export path that includes all imported font files only
  when the user deliberately chooses it.
- Keep raster artwork export pixel-only and independent from font licensing.

## Non-Goals

- No account-synced font library in this slice.
- No Google Fonts catalog API integration or API-key setup in this slice.
- No durable `CanvasDocument` schema migration.
- No renderer algorithm changes.
- No SVG/path-outline text fallback export.

## Acceptance Criteria

- Users can import a Google Fonts family or CSS2 URL from the shared Font
  Library.
- Google Fonts imports carry source/license metadata and render through the
  existing imported font store.
- Normal project package export is safer by default: open fonts can travel,
  unknown local font files do not silently redistribute.
- Users have an explicit all-font package export path for files they have the
  right to distribute.
- Raster export remains pixel-only.
- Existing local font import, `.artifact.json`, `.artifact`, preview, and export
  behavior are not regressed.
- No graph traversal, renderer/export semantics, or thumbnail scheduling change
  is introduced.
