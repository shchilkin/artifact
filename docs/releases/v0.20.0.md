# v0.20.0 Release Notes

Artifact v0.20 is the text workflow release. It makes typography feel like a
first-class creative path while keeping document schema, graph traversal,
thumbnail scheduling, and the normal preview/export path stable.

## Highlights

- Added focused text starts for title, subtitle, label, credits, and poster
  type through the existing text layer model.
- Tuned text defaults and randomization so created type is visible, centered,
  and cover-useful by default.
- Added a curated font registry foundation with Google font loading and canvas
  font readiness.
- Replaced the raw font select with a compact Font Library picker that supports
  categories, search, and live font samples.
- Added a multi-font type starter workflow and removed the destructive screen
  dots treatment from that starter so the fonts remain readable.
- Kept Layers and Nodes aligned through the shared text controls and renderer.
- Added a graceful Clerk/auth fallback so users do not see provider load
  failures as product-facing noise.
- Fixed graph output transparency so transparent node paths do not acquire a
  colored background at the output node.
- Added WebGL-unavailable hardening: if Pixi cannot create a GPU renderer, the
  GPU pass returns the source canvas instead of a blank preview/export frame.

## Validation

- `npm run check`
- `npm run build`
- `npm run test:browser` across Chromium, Firefox, WebKit, mobile Chromium, and
  mobile WebKit. The full browser gate reported `189 passed, 1 skipped`.
- Focused Firefox multi-font browser coverage passed after the GPU fallback.
- GPU fallback unit coverage verifies that a failed Pixi renderer preserves a
  detached nonblank source canvas.
- PR #53 passed quality, browser, Vercel, and GitGuardian checks.

Manual validation:

- The user verified the v0.20 text workflow and follow-up fixes locally before
  release prep.

## Accepted Risks

- Browsers without WebGL/Pixi support now degrade GPU-only effects by returning
  the source canvas unchanged. This is intentional release hardening: it avoids
  blank output, but the effect itself is skipped in that environment.
- Playwright WebKit is the automated Safari-family gate. A manual macOS Safari
  pass is still recommended before a broad public announcement.
- AI generation remains account/API dependent. This release only improves the
  fallback experience when auth/provider scripts fail to load.
- Known dev-server route reload console noise can still appear during local
  development, but it did not produce behavioral failures in the release gate.
