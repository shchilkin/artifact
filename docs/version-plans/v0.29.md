# v0.29 Product Surface Refresh

## Summary

v0.29 should make the outside of Artifact catch up with the editor that now
exists. Layers, Nodes, typography, imported fonts, packages, Add Library, and
guardrails are stronger than the current landing/examples surface suggests.

This slice is about product understanding and visual confidence, not backend
expansion. The goal is that a new user can quickly understand what Artifact is,
open a relevant example, and trust that the examples reflect the real editor.

## Goals

- Refresh the landing page around the current product promise: a dark creative
  editor with Layers, Nodes, typography, procedural sources, effects, packages,
  and export-ready artwork.
- Replace stale examples with examples that demonstrate current workflows.
- Bridge examples and docs so users can move from "what is this" to "how do I
  make something" without reading a long manual.
- Add a small visual confidence baseline for the product surfaces most likely
  to drift.

## Non-Goals

- No server-backed share links.
- No account sync or project sync.
- No AI expansion.
- No renderer, export, graph traversal, thumbnail scheduling, font package
  policy, or document schema changes.
- No corporate SaaS/admin visual redesign.

### 4. Visual Confidence Baseline

- [ ] Add small browser/screenshot coverage for the landing surface.
- [ ] Add small browser/screenshot coverage for the examples surface.
- [ ] Add focused nonblank checks for updated examples.

## Validation

- [ ] `npm run check`
- [ ] `npm run build`
- [ ] `npm run test:browser`
- [ ] Manual browser pass for landing, examples, docs bridge, and generator
  handoff.
- [ ] `npm run perf:node-editor` only if editor canvas behavior, React Flow
  wiring, thumbnail scheduling, or render behavior changes.

## Acceptance Criteria

- Landing communicates the real current product within the first viewport.
- Examples reflect current Layers, Nodes, typography, texture/print, and export
  workflows.
- A new user has a clear path from landing to examples to generator/docs.
- Product surfaces gain a small, useful regression baseline.
- Editor/render/export behavior remains unchanged.
